## Purpose

## Completed Work

### 1. Fixed: Schedule Lockdown Status Not Changing
**Problem**: Frontend lockdown button only updated `lockdownDate` but didn't change the schedule status
**Solution**: Modified `handleLockdown` function in `schedules/index.tsx` to call the correct status change API
**Location**: `apps/web/src/pages/trains/schedules/index.tsx:428`

### 2. Added: Requirement Sub-Status Sync on Lockdown
**Feature**: When schedule status changes to `LOCKED_DOWN`, all onboarded requirements' `subStatus` are automatically updated to `FROZEN`
**Location**: `apps/server/src/modules/trains/services/train.service.ts:1141-1151`

### 3. Added: Requirement Status Sync on Release
**Feature**: When schedule status changes to `RELEASED`, all onboarded requirements' `status` are automatically updated to `RELEASED`
**Location**: `apps/server/src/modules/trains/services/train.service.ts:1154-1164`

### 4. Fixed: Date Replacement Logic
**Problem**: Lockdown/release dates were only set when empty
**Solution**: Now always replaces dates with current time when action is performed
**Location**: `apps/server/src/modules/trains/services/train.service.ts:1124-1131`

### 5. Added: Transaction Protection
**Feature**: All status-related updates are wrapped in database transaction for consistency
**Location**: `apps/server/src/modules/trains/services/train.service.ts:1133-1166`

### 6. Updated: Handoff Skill Naming Convention
**Change**: File naming rule updated from `handoff-XXXXXX.md` to `handoff-yyyymmdd-v1.md`
