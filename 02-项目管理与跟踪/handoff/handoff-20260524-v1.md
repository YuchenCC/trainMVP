### 需求池筛选修复 — 核心问题
**根因**：Fastify 默认不支持 `status[]=value` 数组格式，且 React 状态同步有延迟

**修复链路**：
1. **前端** `requirements/index.tsx`：直接从 URL 读取参数调用 API，将状态数组转为逗号分隔字符串
2. **后端路由** `requirements/index.ts`：`oneOf` → `anyOf` 修复 Fastify schema 验证
3. **后端 service** `requirements/service.ts`：支持逗号分隔字符串的参数解析（`split(',')`）
4. **shared 类型** `dashboard.ts`：添加 `sitDate`/`uatDate` 字段
