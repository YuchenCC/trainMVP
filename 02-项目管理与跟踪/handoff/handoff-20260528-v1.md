## Key Decisions

| 决策 | 结论 |
|------|------|
| schema vs service 校验职责 | schema 做范围校验（0~100），service 做归一化兜底（0→1），非法参数直接 reject 不静默吞掉 |
| Prisma 直建用户 vs seed 端点 | 统一用 `/api/auth/seed`，避免 bcrypt 哈希不一致 |
| page/pageSize 边界 | legacy 测试意图是验证 API 鲁棒性，不应删测试 |

## Failure Root Causes

| 失败 | 根因 | 修复 |
|------|------|------|
| 权限矩阵 5 failures | `Operation` 枚举名拼写错误（如 `APPROVE_REQ`/`CONFIRM_TEST` 不存在） | 对照 `shared/src/constants/index.ts` 修正 |
| 健康检查 404 | 路由是 `/api/health` 不是 `/health` | 修正 URL |
| 登录泄露密码测试 | `afterAll` 清理了用户，安全性测试跑在最后 | `beforeEach` 重新 seed |
| US1.5 登录失败 | Prisma 手动 `bcrypt.hash` 与 seed 端点使用的 salt 轮数或实现不同 | 统一走 seed 端点 |
| pageSize=200 失败 | schema `maximum: 100` 在请求到达 service 前就 reject 了 | 测试预期改为 `VALIDATION_ERROR` |
