## 2. 分层测试方案

| 层级 | 分层原则 |
|---|---|
| L1 后端单测 | 本次纳入范围的后端可执行逻辑默认纳入；覆盖状态门禁、权限判断、字段校验、状态变更、乐观锁和审计写入。增量覆盖率和通过率由 `unit-test-governance` 计算和判定。 |
| L1 前端单测建议 | 本轮未纳入前端代码范围；涉及按钮显隐、页面跳转的案例需由前端测试或人工验证补充。 |
| L2 API/JMeter | 在后端 L1 基础上，验证登录鉴权、RBAC 中间件、HTTP 路由、错误响应、状态变更后的接口结果和数据库副作用。 |
| L3 UI 自动化 | 验证发起评审/重新编辑按钮显隐、状态变化后的操作区、自动进入编辑页等页面行为；本轮前端代码未纳入。 |
| M 人工验证 | 本轮不纳入人工验证；UI 场景不创建人工验证 evidence，保留为 L3 自动化或后续范围。 |
