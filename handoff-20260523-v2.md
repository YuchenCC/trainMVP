## Session Summary（本次做了什么）

### 1. 月历组件核心改造（CalendarView.tsx）

**文件**: `release-train/apps/web/src/components/dashboard/CalendarView.tsx`

- **火车下拉框改造**：改为调 `trainService.list()` API 获取火车列表，切换时调 `getScheduleProgress({ trainId })` 重新请求后端，不再前端过滤。
- **移除"全部火车"选项**：下拉框只显示具体火车，初始化自动选中第一个。
- **孤儿事件支持**：纳版/封板/投产日期如果不在班次 `startDate~endDate` 范围内，独立渲染为彩色 Tag，带 Tooltip 悬停。
- **日历样式修复**：隐藏 Ant Design 默认日期数字 `.ant-picker-calendar-date-value`，只保留 cellRender 自定义日期。
- **孤儿事件布局优化**：flex 两列排列，高度与班次条一致（18px），颜色对应事件类型（纳版蓝/封板橙/投产绿）。
- **容器高度动态计算**：根据当天最大 rowIndex 动态设 paddingTop，解决 absolute 班次条与孤儿事件重叠问题。
- **全文件注释补全**：所有 interface、const、函数、关键逻辑都加上注释。

## Key Technical Notes

### CalendarView 数据流
```
初始化 → trainService.list() 加载火车列表 → 自动选中第一个
          ↓
         trainService.getScheduleProgress({ trainId }) 加载该火车班次
          ↓
切换火车 → switchTrain(newTrainId) → 重新调接口 → setAllSchedules
```

### 孤儿事件逻辑
```
getEventsForDate(date) — 不校验事件日期是否在班次区间内
    ↓
cellRender 渲染时：
  - 有班次条的事件 → 画在条内（barEvents）
  - 无班次条的事件 → 独立渲染 Tag（孤儿事件，flex 两列，带 Tooltip）
```

### 样式关键点
- `.ant-picker-calendar-date-value { display: none }` — 隐藏 Ant Design 默认日期
- `.ant-picker-cell-inner` / `.ant-picker-calendar-date` — 左右 padding/margin 清零
- 容器 `paddingTop` 动态计算 — 为 absolute 班次条撑空间
