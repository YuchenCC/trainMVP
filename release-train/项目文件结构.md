## 关键数据流

### AI 需求审查
```
RequirementForm.tsx (handleAIReview)
  → requirementService.reviewData()
    → Axios POST /api/requirements/review (timeout: 180s)
      → requirement-review/service.ts: reviewRequirementData()
        → runLocalReview()          本地规则检查（标题/描述/格式）
        → runAiReview()             Coze AI 审查
          → buildCozePrompt()       构建提示词
          → coze.runWorkflow()      Coze API (stream 处理)
        → 合并结果 → RequirementReviewResult
      → HTTP 200 → ApiResponse
  → setReviewResult() → Modal 展示（Tabs: 问题/建议/验收条件）
```

### 智能纳版
```
SmartOnboard 页面
  → smartOnboardService.suggest()
    → POST /api/smart-onboard/suggest
      → smart-onboard/service.ts: generateOnboardSuggestions()
        → getScheduleCapacity()    班次容量信息
        → getRequirementsForAI()   需求列表
        → detectCycleDependencies() 循环依赖检测
        → coze.runWorkflow(SMART_ONBOARD)  Coze AI 分析
        → 合并 AI 结果 + 后端计算结果
```

### Coze 客户端
```
coze/index.ts
  → CozeClient.runWorkflow(workflow, parameters)
    → CozeSDK.workflows.runs.stream({ workflow_id, parameters })
    → 流式处理: PING → Message(content=JSON) → Done/workflow_finish
    → 解析 output 字段（支持多重嵌套）
    → 错误处理（Error 事件 → 抛出异常）
```

## 环境配置要点

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | PostgreSQL 连接 |
| `JWT_SECRET` | JWT 签名密钥 |
| `COZE_API_KEY` | Coze API 密钥 |
| `COZE_REQUIREMENT_REVIEW_WORKFLOW_ID` | 需求审查工作流 ID |
| `COZE_SMART_ONBOARD_WORKFLOW_ID` | 智能纳版工作流 ID |
| `COZE_BASE_URL` | Coze API 地址（api.coze.cn） |
