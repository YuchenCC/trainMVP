### 3. T5 需求变更智能体架构重新设计

**原方案**: Coze 平台配置智能体 → 发布到飞书 → 手动配置卡片模板

**新方案**: lark-cli event consume + Coze API 直连 + 项目 API

完整流程：
```
用户 @机器人 转发对话
  → ① lark-cli event consume im.message.receive_v1（接收消息）
  → ② 本地脚本解析消息
  → ③ 调 Coze API workflows.runs.stream()（复用现有 @coze/api SDK）
     → Coze Agent 工作流调项目 API /api/plugin/requirements/{id} 查原始需求
     → Agent 分析变更内容，返回变更摘要+影响评估
  → ④ lark-cli im +messages-send --card（发飞书交互式卡片）
  → ⑤ 用户点击确认/拒绝
  → ⑥ lark-cli event consume（按钮回调）
  → ⑦ 调项目 API POST /api/plugin/change-requests（记录变更单）
  → ⑧ lark-cli im +messages-send（发送确认结果通知）
```
