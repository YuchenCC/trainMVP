```markdown
# {scope} 测试策略与覆盖分析报告

## 1. 分析范围

| 项目 | 内容 |
|---|---|
| 分析范围 |  |
| 测试案例来源 |  |

| 代码范围 |  |

| L1 前端单测建议 | 仅条件触发建议，不是默认硬门禁 | 表单校验、权限显隐、分页排序筛选、金额/日期计算、hook/composable、工具函数、状态管理 | 按项目已有 skill 动态推荐 |
```

## 二、没有测试案例文档时

不生成正式主报告。测试案例文档是本 skill 的必需输入。

如果没有已确认测试案例文档，只输出以下待确认提示，不进入覆盖分析：

```markdown
# {scope} 测试案例缺失提醒

本次测试策略分析未开始，因为缺少已确认测试案例文档。

| 项目 | 内容 |
|---|---|
| 分析范围 |  |
| 已检查位置 |  |
| 候选测试案例 |  |
| 需要用户提供 | 测试案例文档路径，或确认候选文件适用于本次范围 |

请补充测试案例文档后重新执行 test-strategy-planner。
```
