# 测试策略与覆盖分析输出模板

## 一、有测试案例文档时

### 主报告：`requirement-{scope}-测试策略与覆盖分析.md`

```markdown
# {scope} 测试策略与覆盖分析报告

## 1. 分析范围和口径说明

| 项目 | 内容 |
|---|---|
| 分析范围 |  |
| 测试案例来源 |  |
| 需求/设计来源 |  |
| 代码范围 |  |
| 现有测试资产 |  |
| 口径差异 | 无 / 有，说明差异 |

## 2. 覆盖结论

| 范围 | 案例数 | 已覆盖 | 部分覆盖 | 缺失 | 主要缺口 |
|---|---:|---:|---:|---:|---|

## 3. 单元测试门禁策略

| 项目 | 策略 | 说明 |
|---|---|---|
| 增量覆盖率 | > 80% | L2/API、L3/UI、人工验证不能抵扣 |
| 通过率 | 100% | 失败用例必须修复或说明阻断 |
| 优先覆盖 | 纯函数、字段聚合、分页、状态判断、权限矩阵、异常分支 | 必要时建议抽 logic/helper 文件 |
| 后续判断 | 交给 `unit-test-governance` | 本 skill 只规划不判定门禁 |

## 4. 需求/测试案例覆盖矩阵

| 需求点/案例组 | 测试案例 | 推荐层级 | 当前覆盖判断 | 待补内容 | 优先级 |
|---|---|---|---|---|---|

## 5. 代码影响测试矩阵

| 文件/模块 | 影响逻辑 | 影响案例 | 推荐层级 | 必须覆盖原因 | 后续 skill |
|---|---|---|---|---|---|

## 6. 分层测试策略

| 层级 | 是否需要 | 覆盖重点 | 推荐工具 | 交付物 |
|---|---|---|---|---|

## 7. TODO 清单

| 优先级 | TODO | 推荐落地 |
|---|---|---|

## 8. 现有测试资产判断

| 现有测试文件 | 可复用覆盖 | 不足 |
|---|---|---|

## 9. 后续执行建议

| 顺序 | Skill | 输入 | 预期输出 |
|---|---|---|---|

## 10. UI 自动化替代和人工验证证据

| UI 场景 | 替代方式 | 是否询问用户 | evidence 目录 | 必须证据 | 风险 |
|---|---|---|---|---|---|

## 11. 风险和前置条件

| 项目 | 说明 | 影响 | 建议 |
|---|---|---|---|
```

## 二、无测试案例文档时

进入本模式前，必须已经询问用户测试案例文档位置，并得到用户明确确认“没有测试案例文档”。报告中要写明该确认来源。

### 主报告：`requirement-{scope}-test-strategy.md`

```markdown
# {scope} 测试策略规划报告

## 1. 策略摘要

| 项目 | 结果 |
|---|---|
| 变更范围 |  |
| 涉及需求 |  |
| 涉及后端模块 |  |
| 涉及前端模块 |  |
| 测试案例文档 | 用户已确认没有测试案例文档 |
| 推荐测试层级 | L1 优先，L2 其次，L3/M 最后 |
| 是否需要单测门禁 | 增量覆盖率 > 80%，通过率 100% |
| 是否需要接口自动化 |  |
| 是否需要前端/E2E |  |
| 是否允许人工验证替代 UI 自动化 |  |
| 是否已创建 evidence 目录 | 是 / 否，路径 |
| 建议后续 skill |  |

## 2. 单元测试门禁策略

| 项目 | 策略 | 说明 |
|---|---|---|
| 增量覆盖率 | > 80% | L2/L3/M 不能抵扣 |
| 通过率 | 100% | 失败用例必须修复或说明阻断 |
| 后续判断 | 交给 `unit-test-governance` | 本 skill 只规划不判定门禁 |

## 3. 需求覆盖规划矩阵

| 需求点 | 后端影响 | 前端影响 | L1 单测建议 | L2 API 建议 | L3/UI 或人工验证建议 | 优先级 | 风险 |

## 4. 代码影响测试矩阵

|---|---|---|---|---|---|

## 5. UI 自动化替代和人工验证证据

| UI 场景 | 替代方式 | 是否询问用户 | evidence 目录 | 必须证据 | 风险 |
|---|---|---|---|---|---|

## 6. 后续执行建议

| 顺序 | Skill | 输入 | 预期输出 |
|---|---|---|---|

## 7. 风险和前置条件

| 项目 | 说明 | 影响 | 建议 |
|---|---|---|---|
```
