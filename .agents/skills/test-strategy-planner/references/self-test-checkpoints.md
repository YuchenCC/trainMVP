# 自测检查点测试策略（test-strategy-planner 快照）

> 本文件完整复制自 `03-需求与设计/共享/自测检查点-测试策略.md`，用于让 skill references 自包含，避免共享源移动或缺失后丢失规则。

## 本 skill 使用规则

- 作为横向补充检查点来源，不替代测试案例文档。
- 测试案例仍是覆盖分析主线；从本文件发现的问题只能标为“补充风险”或“补充 TODO”。
- 输出时只列本次需求、代码范围和测试案例相关的检查点；不相关内容标记为不适用或不输出。
- 适用示例：API、分页、权限、状态流转、前端交互、数据库变更、文件、定时任务、配置、安全。

# 自测检查点测试策略

> 本策略文档将《自测检查点.md》中的每个检查点映射到具体的测试层次（L1/L2/L3/M/Perf）、优先级、自动化方式和验证方法。

## 一、交互类

### 1. 接口

| 检查点 | 优先级 | 测试层次 | 自动化程度 | 验证方式 | 关联测试类型 |
|--------|--------|----------|------------|----------|--------------|
| 前后端校验规则是否一致，字段关联、校验规则（必填/格式/范围）（如前端允许 11 位手机号，后端仅允许 10 位） | P0 | L2 | 高 | API 参数校验测试，对比前后端校验规则文档 | 接口契约测试 |
| 调用接口需配置合理的超时时间 | P1 | L2 | 中 | 响应时间监控，模拟网络延迟验证超时处理 | 性能测试、接口测试 |
| 报文格式/长度/加密规则 | P0 | L2 | 高 | Schema 校验测试，加密解密验证 | 接口契约测试、安全测试 |
| 防重发，如 Nonce、Timestamp、签名验证加频率限制 | P0 | L2 | 高 | 重复请求测试，验证签名机制 | 安全测试、接口测试 |
| 调用频率限制，限流/熔断/开关等 | P1 | L2 + Perf | 中 | 压测 + 限流规则验证 | 性能测试、稳定性测试 |
| 正常流程测试：确保接口在标准输入下返回正确的输出 | P0 | L2 | 高 | API 契约测试，标准输入验证 | 接口测试 |
| 边界条件测试：测试接口在参数取最大值、最小值、空值、特殊字符等极限情况下的表现 | P0 | L2 | 高 | 参数边界值测试用例 | 接口测试 |
| 错误处理测试：与提供方系统确认接口返回的特殊异常码（包括查无记录等），验证接口返回的各种错误码，有做出相应的正确处理逻辑 | P0 | L2 | 高 | 异常码测试用例，模拟各种错误场景 | 接口测试 |
| 重试机制测试：验证接口是否能够正确重试 | P1 | L2 | 高 | 失败重试测试，模拟网络抖动 | 可靠性测试 |

## 附录：优先级定义

| 优先级 | 定义 | 覆盖要求 |
|--------|------|----------|
| **P0** | 核心功能、安全、交易一致性，必须覆盖 | 100% 自动化覆盖 |
| **P1** | 重要功能、性能、配置，建议覆盖 | 80% 自动化 + 20% 人工 |
| **P2** | 次要功能、运维、混沌工程，按需覆盖 | 50% 自动化 + 50% 人工 |

## 附录：测试层次定义

| 层次 | 定义 | 工具/方式 |
|------|------|-----------|
| **L1** | 单元测试，验证单个函数/方法的逻辑正确性 | vitest/jest |
| **L2** | API 测试，验证接口契约、参数校验、权限控制 | supertest/axios |
| **L3** | UI 自动化测试，验证用户交互流程 | Playwright/Cypress |
| **M** | 人工验证，需要人工审查、截图、DBA 审查等 | 代码审查、人工测试 |
| **Perf** | 性能测试，验证系统在高负载下的表现 | k6/artillery |
