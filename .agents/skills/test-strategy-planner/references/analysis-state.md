# 分析工作状态与续跑规则

本文件定义跨模块的分析工作状态。它保留模块 B 的案例策略、模块 C 的自测检查点白盒审查和模块 D 的测试资产证据，避免上下文压力导致跳过分析直接生成正式报告。

工作状态不是主报告、附件、正式覆盖结论或审批材料；只有模块 E 才能生成正式报告。

**模块 B -> C -> D -> E 的状态模型始终存在。**区别仅在于状态保存在当前对话内，还是持久化为工作状态文件：小范围、上下文充足时保持内存状态并顺序交接；满足持久化触发条件时才创建工作状态文件。

## 1. 创建时机与路径

模块 A 完成范围确认后，不创建工作状态文件。模块 B 形成策略中间结论后，先在当前对话维护工作状态；只有满足下列任一条件时才创建并持续更新工作状态文件：

- 当前上下文接近上限、需要中断或预计无法在当前上下文完成模块 C 或 D。
- 范围较大：跨前后端、超过 10 个相关文件、超过 1500 行相关代码、超过 5 个接口、超过 3 个页面/角色/状态流转，或测试案例超过 10 条。
- 包含多个可闭环案例组，需要分批分析。
- 用户要求阶段性审查、保留中间结果、跨会话续跑或交接。

小范围且上下文充足时，不创建中间文件：模块 B 的内存工作状态依次交给 C、D，D 完成后 E 渲染正式报告。

需要持久化时，使用：

```text
reports/test-strategy/.work/{scope}-analysis-state-v{n}.md
```

- `{scope}` 与正式报告命名规则一致。
- `{n}` 与本轮分析版本一致；重新分析时递增，不覆盖旧状态。
- 用户明确要求不保留中间文件时，仅在当前上下文维护相同结构；上下文压力出现时，必须先询问是否允许创建检查点，不能直接出正式报告。

## 2. 模块 B 写入内容

无论状态在内存还是文件中，模块 B 只写已有测试案例的策略部分，至少包含：

- 已确认范围、测试案例/需求/设计/逻辑代码来源和口径差异。
- 每个已完成案例或案例组的测试目标、推荐层级、分层理由、内部逻辑代码锚点、补充风险和优先级。
- 人工验证候选及用户确认状态。
- 未完成案例、未读逻辑代码和下一步读取计划。

模块 B 不得写入自测检查点完整遍历、已有测试资产实际层级、测试资产证据或“已覆盖/部分覆盖/缺失”的最终覆盖结论。

## 3. 模块 C 补充内容

模块 C 在同一工作状态中补充自测检查点白盒审查，不改写模块 B 的案例策略字段：

- 共享检查点全部叶子项的原始类别路径、原文、优先级和测试层次。
- 每个叶子项的代码证据、涉及/不涉及/待确认判定和理由。
- 涉及/待确认项与已有测试案例的映射和案例覆盖判断。
- 未覆盖或部分覆盖项的补充案例建议、建议层级/理由、风险和 TODO。

模块 C 不得因已有测试资产、工具、测试数量或提交范围改写检查点的涉及判定、原文测试层次、案例覆盖判断或模块 B 的推荐层级。

## 4. 模块 D 补充内容

模块 D 在同一工作状态中补充测试资产证据，不改写模块 B/C 的策略和白盒审查字段：

- 每个既有测试案例的已有测试资产实际层级、文件/测试名称、断言或人工证据、覆盖结论、缺口和 TODO。
- 每个涉及/待确认自测检查点的已有测试资产实际层级、证据和当前证据状态。
- 覆盖统计、可复用资产、未读测试资产和风险。

模块 D 不得因已有资产、工具、测试数量或提交范围改写模块 C 的涉及判定、原文测试层次、代码证据、案例覆盖判断或补充案例建议。

## 5. 模块 E 消费规则

模块 E 读取完成的内存工作状态或持久化工作状态文件及其引用的源文件，按 `output-template.md` 渲染正式主报告和附件：

| 正式报告内容 | 工作状态来源 |
|---|---|
| 分析范围 | 模块 A 确认记录 |
| 分层测试方案、既有案例推荐层级 | 模块 B |
| 自测检查点原文映射、案例缺口与补充建议 | 模块 C |
| 实际测试资产、逐案例覆盖结论、覆盖现状 | 模块 D |
| TODO、风险、人工验证方案 | 模块 B + C + D |

工作状态存在不等于分析完成。只有所有纳入范围的案例完成模块 B，所有共享检查点叶子项完成模块 C，且模块 D 已完成相关资产核对或明确标为阻塞/排除时，才能进入模块 E。

## 6. 上下文压力与恢复

上下文接近上限时：

1. 在当前可闭环案例或案例组结束处停止，不开始新的大范围读取。
2. 更新工作状态，记录已读来源、事实、完成状态、未完成项、待确认项和下一步。
3. 对用户说明当前停在模块 B、C 或 D，不能生成正式报告；不得声称“已读取所有必要参考文件”。
4. 恢复后先读取工作状态和其中引用的相关源文件，再从未完成模块继续。
5. 不得因为上下文压力跳到模块 E，或用工作状态代替未读取的源文件和测试资产。

推荐回复：

```markdown
当前上下文接近上限。我仍处于模块 {B/C/D}，不能生成正式报告。

已保存分析工作状态：`{path}`
已完成：{案例/范围}
待完成：{案例/范围}
恢复后将从模块 {B/C/D} 继续，并在完成模块 D 后再进入模块 E。
```
