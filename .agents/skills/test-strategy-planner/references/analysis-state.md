# 分析工作状态与续跑规则

本文件定义跨模块的分析工作状态。它保留模块 B 的案例策略和模块 C 的自测检查点白盒审查，避免上下文压力导致跳过分析直接生成正式报告。

工作状态不是审批材料；主报告和附件可以在 B/C 阶段按既定目标路径增量生成，只有模块 D 才能声明最终报告已完成并通过输出自检。工作状态用于模型中断后的断点续跑，不要求每次读取文件都写盘。

**状态模型为 B -> C -> D。**区别仅在于状态保存在当前对话内，还是持久化为工作状态文件：小范围、上下文充足时保持内存状态并顺序交接；满足持久化触发条件时才创建工作状态文件。

## 1. 创建时机与路径

模块 A 完成范围确认后，必须创建最小工作状态文件并记录当前目标路径、输入确认结果和模块状态。之后按下列规则持续更新：

- B、C 每个模块完成后至少更新一次。
- 大范围、多案例组或预计会跨上下文时，每个可闭环案例组完成后更新一次。
- 模型即将中断、上下文接近上限、用户要求阶段性审查、跨会话续跑或交接时，立即更新一次。

除上述阶段边界外，不要求每读取一个文件都写盘。以下条件用于决定是否增加案例组级检查点：

- 范围较大：跨前后端、超过 10 个相关文件、超过 1500 行相关代码、超过 5 个接口、超过 3 个页面/角色/状态流转，或测试案例超过 10 条。
- 包含多个可闭环案例组，需要分批分析。
- 用户要求阶段性审查、保留中间结果、跨会话续跑或交接。

统一使用：

```text
reports/test-strategy/.work/{scope}-analysis-state-v{n}.md
```

- `{scope}` 与正式报告命名规则一致。
- `{n}` 与本轮分析版本一致；重新分析时递增，不覆盖旧状态。
- 用户明确要求不保留中间文件时，仍保留 A/B/C 阶段的最小状态检查点；只可减少案例组级明细，不得取消断点续跑所需的阶段状态。

## 2. 模块 B 写入内容

无论状态在内存还是文件中，模块 B 只写已有测试案例的策略部分，至少包含：

- 已确认范围、测试案例/需求/设计/逻辑代码来源和口径差异。
- 每个已完成案例或案例组的测试目标、推荐层级、分层理由、内部逻辑代码锚点、补充风险和优先级。
- 人工验证候选及用户确认状态。
- 未完成案例、未读逻辑代码和下一步读取计划。

模块 B 完成后更新状态文件，至少记录模块 B 已完成、已写入报告的目标路径和章节、已完成案例、未完成案例及下一步模块 C 计划。

模块 B 不得写入自测检查点完整遍历或覆盖结论。

## 3. 模块 C 补充内容

模块 C 在同一工作状态中补充自测检查点白盒审查，不改写模块 B 的案例策略字段：

- 模块 B 逻辑事实提炼出的行为清单、范围排除清单和范围证据编号。
- 共享检查点全部叶子项的原始类别路径、原文、优先级和测试层次。
- 每个叶子项的代码证据、涉及/不涉及/待确认判定和理由；批量不涉及项记录范围证据编号，候选深审项记录具体代码事实。
- 候选深审项的风险簇，以及涉及/待确认项与既有测试案例的映射和案例覆盖判断。
- 未覆盖或部分覆盖项的补充案例建议、建议层级/理由、风险和 TODO；多个检查点复用建议时记录同一建议编号。

模块 C 完成后更新状态文件，至少记录全量检查点遍历状态、范围证据编号、已完成/未完成风险簇、已写入报告的章节和补充案例建议；下一步直接进入模块 D。

## 4. 模块 D 消费规则

模块 D 读取完成的内存工作状态或持久化工作状态文件，按 `output-template.md` 校验已由 B/C 增量形成的正式主报告和附件：

| 正式报告内容 | 工作状态来源 | 写入时机 |
|---|---|---|
| 分析范围 | 模块 A 确认记录 | B 初始化报告时 |
| 分层测试方案、既有案例推荐层级 | 模块 B | B 完成后立即 |
| 自测检查点原文映射、案例缺口与补充建议 | 模块 C | C 完成后立即 |
| 执行准备、TODO、风险、人工验证方案 | 模块 B + C | 对应阶段增量补充 |

工作状态存在不等于分析完成。报告可以在 B/C 阶段逐步形成；只有所有纳入范围的案例完成模块 B，所有共享检查点叶子项完成模块 C，才能进入模块 D 做最终校验。

## 5. 上下文压力与恢复

上下文接近上限时：

1. 在当前可闭环案例或案例组结束处停止，不开始新的大范围读取。
2. 更新工作状态，记录已读来源、事实、完成状态、未完成项、待确认项和下一步。
3. 对用户说明当前停在模块 B、C 或 D，不能声明最终报告已完成；不得声称“已读取所有必要参考文件”。
4. 恢复后先读取工作状态和已有报告的最近增量章节，再按状态中记录的未完成模块和引用范围补读相关源文件；不得从 A 重新开始。
5. 不得因为上下文压力跳到模块 D，或用工作状态代替未读取的源文件。

推荐回复：

```markdown
当前上下文接近上限。我仍处于模块 {B/C/D}，不能生成正式报告。

已保存分析工作状态：`{path}`
已完成：{案例/范围}
待完成：{案例/范围}
恢复后将从模块 {B/C/D} 继续，并在完成当前模块后进入下一阶段。
```
