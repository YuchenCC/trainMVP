# TestGov Unit Gate

## 用途

用于后端 L1 单元测试治理。后端单测是默认硬门禁。

默认执行人：后端开发。审核人：研发负责人和测试负责人。

前端单测不是默认硬门禁。前端变更只识别是否触发单测建议。

## 后端硬门禁

后端门禁同时满足以下条件才算通过：

```text
后端增量单测覆盖率 > 80%
后端单测通过率 = 100%
```

API 测试、JMeter、UI 自动化、人工验证、前端测试、集成测试、E2E 测试都不能计入后端单测门禁。

## 前端单测条件触发

仅对可隔离复杂前端逻辑建议单测：

| 触发场景 | 结果 |
|---|---|
| 表单校验、字段格式化、权限显隐判断、按钮状态 | 建议补前端单测 |
| 筛选/排序/分页、金额/日期/数量计算 | 应补前端单测 |
| React hook、Vue composable、工具函数、状态管理 | 应补前端单测 |
| 纯展示、简单表格、普通弹窗、文案展示 | 不触发前端单测，走 L3/UI 或人工验证 |

不得因为缺少前端单测而让后端单测门禁失败。

## 工作流程

1. 读取 specs 和 analyzer 报告。
2. 识别后端变更的可执行代码。
3. 写 TODO 前，先搜索已有后端单测。
4. 环境允许时，运行最小安全相关后端单测命令。
5. 解析或引用 JaCoCo / 后端覆盖率证据。
6. 分类变更逻辑：已覆盖、缺失、排除、需要非单测证据、阻塞。
7. 输出后端门禁结果：`通过`、`不通过`、`阻塞`。
8. 如果涉及前端变更，单独输出前端单测触发建议。

## 输出

写入：

```text
reports/testgov/unit-gate-{scope}.md
```

必须包含：

1. 后端单测门禁摘要。
2. 已有后端单测文件和执行证据。
3. 后端增量覆盖率证据。
4. 后端单测通过率证据。
5. 变更逻辑覆盖分类。
6. 缺失单测 TODO 清单。
7. 不适合单测或排除项的风险说明。
8. 前端单测触发建议。
9. 给 final-report 的结论摘要。

未能执行测试时，不得写通过；只能写阻塞，并说明缺失的命令、环境或报告。
## 输出模板规则

生成输出文件前，必须先读取本 skill 自带模板：

```text
references/output-template.md
```

按模板结构输出。没有证据的字段写 `缺失`、`待确认` 或 `不适用`，不得编造结果。
