# TestGov Final Report

## 用途

这是 TestGov 系列最后一步，用于生成管理层可读的最终报告。

默认执行人：测试负责人或测试经理。审核人：项目经理、测试经理、研发负责人。

本 skill 不重新计算、不编造各层结果。必须引用上游报告，缺失证据要明确标记。

## 输入规则

不要重新读取所有原始需求、测试案例或源码。本 skill 只聚合上游摘要和结论。优先读取：

- specs 摘要。
- analyzer 摘要、矩阵、TODO 和风险。
- unit-gate 结果。
- api-jmeter 结果。
- ui-flow 结果。
- manual-evidence summary。

如果上游报告过长，只读取摘要、结论、TODO、风险、证据索引章节。

## 决策规则

- 后端单测门禁失败时，默认阻断正常提测，除非有明确审批人接受风险。
- 前端单测不是默认硬门禁，只报告是否触发建议以及是否处理。
- API/UI/人工证据不能抵扣后端单测门禁。
- 人工验证必须有 `summary.md`；只有触发 DB 证据场景时才要求 DB 截图。
- 没有证据不得写“已覆盖”。只能写 `缺失`、`待补` 或 `用户确认不适用`。

最终结论只能从以下值选择：

```text
可提测
有条件提测
不建议提测
可上线
不建议上线
```

## 输出

写入：

```text
reports/testgov/final-{scope}.md
```

必须包含：

1. 管理层摘要。
2. 范围、分支、commit、环境和证据来源。
3. 按用户故事汇总的需求覆盖情况。
4. 后端单测门禁结果：增量覆盖率、通过率、是否通过。
5. 前端单测触发建议处理情况。
6. API/JMeter 覆盖情况。
7. UI 自动化覆盖情况。
8. 人工验证覆盖情况。
9. 测试执行明细和证据索引。
10. 失败、阻塞、未执行项说明。
11. 未覆盖风险、风险接受人和责任人。
12. 最终建议。

报告必须中文、简洁、结论前置，适合项目经理和测试经理快速判断本次需求和代码的测试覆盖情况。
## 输出模板规则

生成输出文件前，必须先读取本 skill 自带模板：

```text
references/output-template.md
```

按模板结构输出。没有证据的字段写 `缺失`、`待确认` 或 `不适用`，不得编造结果。
