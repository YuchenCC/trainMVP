# TestGov API JMeter

## 用途

用于 L2 接口自动化测试治理。读取 specs、analyzer 报告和 JMeter/API 执行结果，输出接口覆盖情况。

默认执行人：测试人员或后端开发。审核人：测试负责人。

本 skill 不抵扣后端单测门禁，不生成最终管理层报告。

## 适用范围

- 接口契约和字段校验。
- 鉴权、权限、角色访问。
- 入参合法/非法分支。
- 状态变化、幂等、重复提交。
- 关键错误码和异常响应。
- 跨服务接口链路。

## 工作流程

1. 读取 specs 和 analyzer 中的 L2/API 任务。
2. 查找已有 JMeter、Postman、接口自动化脚本或报告。
3. 能安全执行时，运行相关接口自动化；不能执行时引用已有报告并标记缺口。
4. 将接口用例映射到需求、测试案例和代码影响点。
5. 输出已覆盖、部分覆盖、缺失、阻塞和不适用。

## 输出

写入：

```text
reports/testgov/api-jmeter-{scope}.md
```

必须包含：

1. API 覆盖摘要。
2. 执行环境、分支、报告来源。
3. 接口 / 测试案例 / 需求映射表。
4. 通过、失败、阻塞、未执行清单。
5. 缺失 JMeter/API TODO。
6. 对 final-report 的结论摘要。
## 输出模板规则

生成输出文件前，必须先读取本 skill 自带模板：

```text
references/output-template.md
```

按模板结构输出。没有证据的字段写 `缺失`、`待确认` 或 `不适用`，不得编造结果。
