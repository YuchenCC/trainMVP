# 领域模型与状态机详细设计

Status: ready-for-human
Type: HITL

## What to build

完成版本火车需求管理系统的领域模型与需求状态机详细设计，明确需求、版本火车、搭载系统、纳版、团队容量、依赖风险、操作审计之间的关系，并定义每个状态机事件的允许条件、状态变化、伴随副作用和错误规则。

## Acceptance criteria

- [ ] 明确需求状态机事件、前置状态、目标状态和副作用。
- [ ] 覆盖纳版、移除、已纳版变更、投产、回滚、取消需求、完成版本火车等核心行为。
- [ ] 标明哪些规则是硬阻断，哪些是软风险提示。
- [ ] 定义操作审计事件类型和关键摘要字段。

## Blocked by

None - can start immediately
