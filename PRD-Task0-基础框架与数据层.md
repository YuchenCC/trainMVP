## 二、术语定义（全局）

### 2.1 核心术语

| 术语 | 定义 |
|------|------|
| **版本火车** | 固定周期（2/3/4周可配置）的软件发布计划容器，可搭载多个系统 |
| **纳版** | 火车管理员确认已就绪需求进入版本火车交付范围，开始占用团队容量 |
| **已就绪** | 需求信息完整并通过评审，可进入版本规划 |
| **团队容量** | 某系统团队在一个版本火车周期内可承接的工作量点数，是软约束 |
| **依赖风险** | 需求前置依赖未满足、未参车或已取消时产生的纳版风险 |
| **需求变更** | 对已就绪或已纳版（非封板）需求的显式变更，退回草稿 |
| **紧急变更** | 对封板状态需求的变更申请，需项目经理或火车管理员审批 |
| **投产** | 已纳版需求完成上线发布后的需求级状态变化 |
| **回滚** | 已投产需求撤回并回到已就绪状态 |
| **取消需求** | 需求在投产前被终止，已投产需求不能取消 |
| **完成版本火车** | 火车管理员手动确认关闭版本火车 |

### 2.2 状态定义

**需求主状态**：
- `DRAFT` - 草稿
- `PENDING_REVIEW` - 待评审
- `READY` - 已就绪
- `REJECTED` - 已拒绝
- `ONBOARDED` - 已纳版
- `RELEASED` - 已投产
- `CANCELLED` - 已取消

**需求子状态**（仅 ONBOARDED 时）：
- `DEV_IN_PROGRESS` - 开发中
- `SIT_TESTING` - SIT测试
- `UAT_TESTING` - UAT测试
- `FROZEN` - 封板

**火车状态**：
- `PLANNING` - 规划中
- `IN_PROGRESS` - 进行中
- `COMPLETED` - 已完成
- `CANCELLED` - 已取消

## 三、状态流转规则

### 3.1 主线流转

```
DRAFT → PENDING_REVIEW → READY → ONBOARDED → RELEASED
```

### 3.2 分支流转

| 从状态 | 可流转到 | 触发条件 |
|--------|----------|----------|
| DRAFT | PENDING_REVIEW | 发起评审 |
| DRAFT | CANCELLED | 取消需求 |
| PENDING_REVIEW | READY | 评审通过 |
| PENDING_REVIEW | REJECTED | 评审拒绝 |
| PENDING_REVIEW | CANCELLED | 取消需求 |
| READY | ONBOARDED | 纳版搭载 |
| READY | CANCELLED | 取消需求 |
| READY | DRAFT | 需求变更 |
| REJECTED | DRAFT | 重新编辑 |
| REJECTED | CANCELLED | 取消需求 |
| ONBOARDED | READY | 从火车移除 |
| ONBOARDED | RELEASED | 确认投产 |
| ONBOARDED | CANCELLED | 取消需求 |
| ONBOARDED | DRAFT | 需求变更/紧急变更 |
| RELEASED | READY | 回滚 |

### 3.3 子状态流转（ONBOARDED 内部）

```
DEV_IN_PROGRESS → SIT_TESTING → UAT_TESTING → FROZEN → RELEASED
```

**子状态分支**：
- 任意子状态可 → READY（从火车移除）
- 非 FROZEN 子状态可 → DRAFT（需求变更）
- FROZEN 可 → DRAFT（紧急变更，需审批）

### 3.4 关键限制

1. **已投产不能取消**：RELEASED 状态只能回滚到 READY，不能取消
2. **需求变更范围**：READY 或 ONBOARDED（非 FROZEN）
3. **紧急变更范围**：仅 ONBOARDED + FROZEN 状态
4. **变更后流程**：回到 DRAFT 后需重新发起评审

### 4.1 角色定义

| 技术经理 | TECH_MGR | 开发完成确认、点数修改 |

### 4.3 特殊权限规则

- **点数修改**：项目经理或技术经理可修改 READY 状态需求的点数，不需要重新评审
- **已纳版需求**：修改点数必须走需求变更流程
- **数据权限**：BA/PM 只能操作自己归属系统的需求

## 五、依赖风险分级

### 5.1 风险等级

| 等级 | 条件 | 提示方式 | 强制纳版 |
|------|------|----------|----------|
| `warning` | 依赖未纳入本车 | 黄色提示 | 允许，无需确认 |
| `high` | 依赖状态为草稿/待评审/已拒绝 | 橙色警告 | 允许，需确认说明 |
| `critical` | 依赖已取消 | 红色危险 | 允许，需确认说明 |

### 5.2 依赖满足条件

- 依赖需求状态为 `ONBOARDED` 或 `RELEASED` → 依赖已满足 ✅
- 其他状态 → 按上表分级提示
