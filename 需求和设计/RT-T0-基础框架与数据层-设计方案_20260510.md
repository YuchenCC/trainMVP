## 五、数据库Schema设计

### 5.1 ER关系图

```
User ──(1:N)──► Requirement ──(N:1)──► System
  │                    │
  │                    ├──(1:N)──► StatusLog
  │                    │
  │                    ├──(1:N)──► RequirementDependency（双向依赖关系）
  │                    │
  │                    └──(1:1)──► EmergencyChange
  │
  └──(1:N)──► Train ──(1:N)──► TrainSystem ◄──(N:1)── System
                    │
                    └──(1:N)──► Requirement

System ◄──(N:M)──► SystemMember ──(N:1)──► User
```

> **v1.2 变更说明**：Requirement 新增 RequirementDependency 独立模型替代 Json 字段，新增 reqCode/version/reqType/sourceChannel/proposedAt 字段，baOwnerId→baId，priority 从 Int 改为 Priority 枚举，StatusLog 新增 operationType 字段。

### 5.2 完整Schema

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ========== 用户 ==========
model User {
  id          String   @id @default(cuid())       // 用户ID
  username    String   @unique                      // 用户名
  displayName String                                // 显示名称
  email       String   @unique                      // 邮箱
  password    String?                               // 密码（本地开发用，生产走SSO）
  role        Role     @default(BA)                 // 角色
  ssoId       String?  @unique                      // SSO对接ID
  createdAt   DateTime @default(now())              // 创建时间
  updatedAt   DateTime @updatedAt                   // 更新时间

  ownedRequirements   Requirement[]  @relation("BA")        // 作为业务归属人的需求
  pmRequirements      Requirement[]  @relation("PM")        // 作为产品经理的需求
  createdRequirements Requirement[]  @relation("Creator")   // 创建的需求
  createdTrains       Train[]        @relation("TrainCreator") // 创建的火车
  statusLogs          StatusLog[]    @relation("StatusOperator") // 操作的状态日志
  emergencyApprovals  EmergencyChange[] @relation("Approver")     // 审批的紧急变更
  systemMembers       SystemMember[]                       // 所属系统成员
}

enum Role {
  BA             // 业务BA
  PM             // 产品经理
  PROJECT_MGR    // 项目经理
  TECH_MGR       // 技术经理
  TEST_MGR       // 测试经理
  TRAIN_ADMIN    // 火车管理员
  SUPER_ADMIN    // 超级管理员
}

// ========== 系统 ==========
model System {
  id          String   @id @default(cuid())       // 系统ID
  name        String   @unique                      // 系统名称
  description String?                               // 系统描述
  createdAt   DateTime @default(now())              // 创建时间
  updatedAt   DateTime @updatedAt                   // 更新时间

  requirements Requirement[]                         // 系统下的需求
  trainSystems TrainSystem[]                        // 系统参与的火车
  members      SystemMember[]                      // 系统成员
}

// ========== 系统成员 ==========
model SystemMember {
  id       String     @id @default(cuid())         // 成员ID
  systemId String                                   // 所属系统ID
  userId   String                                   // 用户ID
  role     SystemRole @default(DEV)                 // 成员角色

  system   System     @relation(fields: [systemId], references: [id], onDelete: Cascade)
  user     User       @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([systemId, userId, role])               // 同一系统同一用户同一角色唯一
}

enum SystemRole {
  BA            // 业务BA
  PM            // 产品经理
  PROJECT_MGR   // 项目经理
  TECH_MGR      // 技术经理
  TEST_MGR      // 测试经理
  DEV           // 开发人员
  QA            // 测试人员
}

// ========== 需求 ==========
model Requirement {
  id              String        @id @default(cuid())       // 需求ID，自动生成的CUID
  reqCode         String        @unique                      // 需求编号，格式REQ-{年份}-{4位序号}，全局唯一
  title           String                                    // 需求标题，简洁描述核心内容
  description     String        @db.Text                     // 需求描述，HTML富文本（存储时已做XSS过滤）
  systemId        String                                    // 归属系统ID，必填
  priority        Priority                                    // 优先级 P0/P1/P2/P3
  storyPoints     Int                                         // 工作量点数 1-100
  baId            String                                    // 业务归属人ID
  pmId            String?                                   // 产品经理ID（可选）
  creatorId       String                                    // 创建人ID
  status          ReqStatus     @default(DRAFT)             // 主状态
  subStatus       ReqSubStatus?                              // 子状态（仅已纳版有值）
  trainId         String?                                   // 所属火车ID（可选）
  version         Int           @default(1)                  // 乐观锁版本号
  reqType         ReqType?                                   // 需求类型（可选）
  sourceChannel   SourceChannel?                             // 来源渠道（可选）
  proposedAt      DateTime      @default(now())              // 提出时间
  createdAt       DateTime      @default(now())              // 创建时间
  updatedAt       DateTime      @updatedAt                   // 更新时间

  system          System        @relation(fields: [systemId], references: [id])
  ba              User          @relation("BA", fields: [baId], references: [id])
  pm              User?         @relation("PM", fields: [pmId], references: [id])
  creator         User          @relation("Creator", fields: [creatorId], references: [id])
  train           Train?        @relation(fields: [trainId], references: [id])

  dependants      RequirementDependency[] @relation("DependantRequirement")
  dependencies    RequirementDependency[] @relation("DependencyRequirement")

  statusLogs      StatusLog[]
  emergencyChanges EmergencyChange[]

  @@index([status, subStatus])
  @@index([systemId])
  @@index([baId])
  @@index([trainId])
  @@index([priority])
}

// ========== 需求依赖关系 ==========
model RequirementDependency {
  id             String       @id @default(cuid())
  dependantId    String                                   // 依赖方需求ID
  dependencyId   String                                   // 被依赖方需求ID

  dependant      Requirement  @relation("DependantRequirement", fields: [dependantId], references: [id], onDelete: Cascade)
  dependency     Requirement  @relation("DependencyRequirement", fields: [dependencyId], references: [id], onDelete: Cascade)

  @@unique([dependantId, dependencyId])
  @@index([dependencyId])
}

// ========== 优先级枚举 ==========
enum Priority {
  P0   // 最高优先级（线上故障/业务阻断）
  P1   // 高优先级（核心功能/重要需求）
  P2   // 中优先级（常规需求）
  P3   // 低优先级（优化/体验类）
}

// ========== 需求类型枚举 ==========
enum ReqType {
  NEW_FEATURE   // 新功能
  OPTIMIZATION  // 优化
  BUG           // 缺陷
}

// ========== 来源渠道枚举 ==========
enum SourceChannel {
  BUSINESS      // 业务提出
  USER_FEEDBACK // 用户反馈
  DATA_ANALYSIS // 数据分析
  COMPETITOR    // 竞品分析
}

enum ReqStatus {
  DRAFT           // 草稿
  PENDING_REVIEW  // 待评审
  READY           // 已就绪
  REJECTED        // 已拒绝
  ONBOARDED       // 已纳版
  RELEASED        // 已投产
  CANCELLED       // 已取消
}

enum ReqSubStatus {
  DEV_IN_PROGRESS  // 开发中
  SIT_TESTING      // SIT测试
  UAT_TESTING      // UAT测试
  FROZEN           // 封板
}

// ========== 状态变更记录 ==========
model StatusLog {
  id             String        @id @default(cuid())         // 日志ID
  requirementId  String                                    // 需求ID
  operationType  OperationType                              // 操作类型
  fromStatus     ReqStatus?                                // 变更前状态
  toStatus       ReqStatus                                 // 变更后状态
  fromSubStatus  ReqSubStatus?                             // 变更前子状态
  toSubStatus    ReqSubStatus?                             // 变更后子状态
  reason         String?       @db.Text                    // 变更原因
  operatorId     String                                    // 操作人ID
  operator       User          @relation("StatusOperator", fields: [operatorId], references: [id])
  createdAt      DateTime      @default(now())              // 操作时间

  requirement    Requirement   @relation(fields: [requirementId], references: [id], onDelete: Cascade)

  @@index([requirementId, createdAt(sort: Desc)])
}

// ========== 操作类型枚举 ==========
enum OperationType {
  CREATE                // 创建需求
  UPDATE                // 编辑需求（草稿状态）
  SUBMIT_REVIEW         // 发起评审
  REVIEW_PASS           // 评审通过
  REVIEW_REJECT         // 评审拒绝
  RE_EDIT               // 重新编辑（已拒绝→草稿）
  CANCEL                // 取消需求
  CHANGE_REQUIREMENT    // 需求变更
  EMERGENCY_CHANGE      // 紧急变更
  DEV_COMPLETE          // 开发完成
  SIT_PASS              // SIT通过
  UAT_PASS              // UAT通过
  BATCH_SUBMIT_REVIEW   // 批量发起评审
  BATCH_CANCEL          // 批量取消
  BATCH_CHANGE_PRIORITY // 批量修改优先级
  ONBOARD               // 纳版
  REMOVE                // 从火车移除
  RELEASE               // 投产
  ROLLBACK              // 回滚
}

// ========== 版本火车 ==========
model Train {
  id            String       @id @default(cuid())         // 火车ID
  name          String                                     // 火车名称
  cycleWeeks    Int          @default(4)                   // 版本周期（周）
  status        TrainStatus  @default(PLANNING)            // 火车状态
  description   String?      @db.Text                     // 火车描述

  startDate     DateTime                                   // 开始日期
  endDate       DateTime                                   // 结束日期
  boardingDate  DateTime?                                  // 统一纳版日（自动计算）
  lockdownDate  DateTime?                                  // 统一封板日（自动计算）
  releaseDate   DateTime?                                  // 统一投产日（自动计算）

  createdById   String                                     // 创建人ID
  createdBy     User         @relation("TrainCreator", fields: [createdById], references: [id]) // 创建人
  createdAt     DateTime     @default(now())              // 创建时间
  updatedAt     DateTime     @updatedAt                   // 更新时间

  requirements  Requirement[]                            // 火车下的需求
  trainSystems  TrainSystem[]                            // 火车-系统关联

  @@index([status])                                      // 按火车状态筛选
  @@index([startDate])                                   // 按时间范围查询火车
}

enum TrainStatus {
  PLANNING      // 规划中
  IN_PROGRESS   // 进行中
  COMPLETED     // 已完成
  CANCELLED     // 已取消
}

// ========== 火车-系统关联 ==========
model TrainSystem {
  id             String  @id @default(cuid())             // 关联ID
  trainId        String                                     // 火车ID
  systemId       String                                     // 系统ID
  capacityPoints Int     @default(0)                       // 容量点数（系统可承接的最大工作量）
  usedPoints     Int     @default(0)                       // 已用点数（已分配的需求工作量）

  train          Train   @relation(fields: [trainId], references: [id], onDelete: Cascade) // 所属火车
  system         System  @relation(fields: [systemId], references: [id], onDelete: Cascade) // 所属系统

  @@unique([trainId, systemId])                            // 同一火车同一系统唯一
}

// ========== 紧急变更审批 ==========
model EmergencyChange {
  id             String          @id @default(cuid())       // 变更ID
  requirementId  String                                     // 关联需求ID
  requirement    Requirement     @relation(fields: [requirementId], references: [id], onDelete: Cascade) // 关联需求
  urgency        UrgencyLevel                               // 紧急程度
  reason         String          @db.Text                   // 变更原因
  status         ApprovalStatus  @default(PENDING)          // 审批状态
  approverId     String?                                    // 审批人ID
  approver       User?            @relation("Approver", fields: [approverId], references: [id]) // 审批人
  approvedAt     DateTime?                                  // 审批时间
  rejectReason   String?         @db.Text                   // 驳回原因
  createdAt      DateTime        @default(now())            // 创建时间

  @@index([status, approverId])                           // 审批人查看待审批列表
}

enum UrgencyLevel {
  P0              // 最高紧急（线上故障）
  P1              // 高紧急（业务阻断）
}

enum ApprovalStatus {
  PENDING         // 待审批
  APPROVED        // 已通过
  REJECTED        // 已驳回
}
```

## 七、RBAC权限矩阵

| 操作 | BA | 产品经理 | 项目经理 | 技术经理 | 测试经理 | 火车管理员 | 超级管理员 |
|------|:--:|:-------:|:-------:|:------:|:------:|:--------:|:--------:|
| 创建需求 | ✅(归属系统) | ✅(归属系统) | ✅ | ❌ | ❌ | ❌ | ✅ |
| 编辑需求 | ✅(草稿不限归属人) | ✅(草稿不限归属人) | ✅(草稿不限归属人) | ❌ | ❌ | ❌ | ✅ |
| 发起评审 | ✅(草稿不限归属人) | ✅(草稿不限归属人) | ✅(草稿不限归属人) | ❌ | ❌ | ❌ | ✅ |
| 评审通过/拒绝 | ❌ | ✅ | ✅ | ✅ | ❌ | ❌ | ✅ |
| 取消需求 | ✅(归属人) | ❌ | ✅ | ❌ | ❌ | ✅ | ✅ |
| 需求变更(已就绪/已纳版非封板) | ✅(归属人) | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| 紧急变更(封板) | ✅(归属人) | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| 紧急变更审批 | ❌ | ❌ | ✅ | ❌ | ❌ | ✅ | ✅ |
| 开发完成 | ❌ | ❌ | ❌ | ✅ | ❌ | ❌ | ✅ |
| SIT通过 | ❌ | ❌ | ❌ | ❌ | ✅ | ❌ | ✅ |
| UAT通过(封板) | ✅(归属人) | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ |
| 创建火车 | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| 纳版/移除/投产 | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| 回滚 | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| 完成版本火车 | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |

> **v1.2 变更说明**：
> - 创建需求：项目经理也可创建
> - 编辑需求/发起评审：草稿状态 BA/PM/PROJECT_MGR 均可操作，不限归属人
> - 取消需求：项目经理也可取消

> **说明**：
> - 已投产需求不能取消，只能通过回滚操作撤回
> - 需求变更：已就绪/已纳版（开发中/SIT测试/UAT测试）→ 草稿
> - 紧急变更：封板状态 → 草稿（需项目经理或火车管理员审批）
> - 工作量点数修改：项目经理或技术经理可修改，不需要重新评审
