### 2.2 状态流转确认

**主线状态**: 草稿 → 待评审 → 已就绪 → 已纳版 → 已投产 → 已取消

**已纳版子状态**: 开发中 → SIT测试 → UAT测试 → 封板

**分支流转**:
- 待评审 → 已拒绝（可重新编辑后再次提交）
- 已就绪 → 草稿（需求变更）
- 已纳版（非封板）→ 草稿（需求变更）
- 封板 → 草稿（紧急变更，需审批）

> **v1.1变更**: Task 1 不包含 `已投产 → 已就绪（回滚）` 路径，该流转在 Task 2 实现

### 3.2 状态流转校验规则

```typescript
// 状态流转图定义（Task 1不包含RELEASED→READY回滚路径）
const STATUS_TRANSITIONS: Record<ReqStatus, ReqStatus[]> = {
  DRAFT: ['PENDING_REVIEW', 'CANCELLED'],
  PENDING_REVIEW: ['READY', 'REJECTED', 'CANCELLED'],
  READY: ['ONBOARDED', 'CANCELLED', 'DRAFT'], // 需求变更
  REJECTED: ['DRAFT', 'CANCELLED'],
  ONBOARDED: ['READY', 'CANCELLED', 'DRAFT'], // 需求变更/紧急变更
  RELEASED: [], // Task 1无操作，Task 2加回滚
  CANCELLED: [],
};

// 子状态流转（仅ONBOARDED状态下）
const SUB_STATUS_TRANSITIONS: Record<ReqSubStatus, ReqSubStatus[]> = {
  DEV_IN_PROGRESS: ['SIT_TESTING', 'READY', 'DRAFT'],
  SIT_TESTING: ['UAT_TESTING', 'READY', 'DRAFT'],
  UAT_TESTING: ['FROZEN', 'READY', 'DRAFT'],
  FROZEN: ['DRAFT'], // 紧急变更（投产在Task 2）
};

// 状态流转校验函数
function canTransition(
  currentStatus: ReqStatus,
  targetStatus: ReqStatus,
  currentSubStatus?: ReqSubStatus,
  targetSubStatus?: ReqSubStatus
): boolean {
  if (!STATUS_TRANSITIONS[currentStatus]?.includes(targetStatus)) {
    return false;
  }
  
  if (currentStatus === 'ONBOARDED' && targetStatus === 'ONBOARDED') {
    if (!currentSubStatus || !targetSubStatus) return false;
    return SUB_STATUS_TRANSITIONS[currentSubStatus]?.includes(targetSubStatus);
  }
  
  return true;
}

// 操作权限校验
const OPERATION_PERMISSIONS: Record<OperationType, Role[]> = {
  CREATE: [Role.BA, Role.PM, Role.TRAIN_ADMIN, Role.SUPER_ADMIN],
  UPDATE: [Role.BA, Role.PM, Role.TRAIN_ADMIN, Role.SUPER_ADMIN],
```

### 3.3 需求变更数据重置规则

需求变更（普通变更/紧急变更）退回草稿时，字段重置规则：

| 字段 | 处理方式 |
|------|----------|
| subStatus | **清空**（设为null） |
| trainId | **清空**（设为null，从火车移除） |
| 依赖关系 | **保留**（变更后依赖可能不变，保留有助于追溯） |
| storyPoints | **保留但允许编辑**（变更可能影响工作量） |
| 其他字段 | 保留不变 |

### 6.1 循环依赖检测

```typescript
// 检测需求依赖是否存在循环
function hasCircularDependency(
  dependantId: string,
  dependencyId: string,
  existingDeps: Map<string, string[]>
): boolean {
  // DFS检测从dependencyId出发是否能回到dependantId
  const visited = new Set<string>();
  const stack = [dependencyId];
  
  while (stack.length > 0) {
    const current = stack.pop()!;
    if (current === dependantId) return true;
    if (visited.has(current)) continue;
    visited.add(current);
    
    const deps = existingDeps.get(current) || [];
    stack.push(...deps);
  }
  
  return false;
}
```

```typescript
function checkDependencyRisk(dependencyStatus: ReqStatus): {

  }
```

```typescript
async function updateWithOptimisticLock(
  prisma: PrismaClient,
  id: string,
  version: number,
  updateData: Partial<Requirement>
): Promise<Requirement> {
  const result = await prisma.requirement.updateMany({
    where: { id, version, deletedAt: null },
    data: { ...updateData, version: { increment: 1 } },
  });
  
  if (result.count === 0) {

  }
  
  return prisma.requirement.findUniqueOrThrow({ where: { id } });
}
```

```typescript
// 生成需求编号 REQ-{年份}-{4位序号}
async function generateReqCode(prisma: PrismaClient): Promise<string> {
  const year = new Date().getFullYear();
  const prefix = `REQ-${year}-`;
  
  const lastReq = await prisma.requirement.findFirst({
    where: { reqCode: { startsWith: prefix } },
    orderBy: { reqCode: 'desc' },
    select: { reqCode: true },
  });
  
  let nextSeq = 1;
  if (lastReq) {
    const lastSeq = parseInt(lastReq.reqCode.split('-')[2], 10);
    nextSeq = lastSeq + 1;
  }
  
  return `${prefix}${nextSeq.toString().padStart(4, '0')}`;
}
```
