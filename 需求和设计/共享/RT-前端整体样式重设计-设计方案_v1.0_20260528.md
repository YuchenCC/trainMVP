## 1. 设计结论

本次前端样式重设计采用 **白底调度工作台** 方向。

系统保留现有 React + Ant Design 技术栈，不推翻业务流程、不修改后端接口、不调整权限矩阵。通过统一主题、布局规范、基础组件和核心页面样式，把当前分散的 Ant Design 默认页面收敛为一套干净、专业、高信息密度的版本火车需求管理系统。

---

## 2. 当前问题

### 2.1 样式分散

当前前端样式主要写在 TSX 的 `style`、`bodyStyle`、`styles` 中，缺少统一的主题文件和全局样式层。

影响：

- 同类组件在不同页面表现不一致
- 颜色、圆角、阴影、间距难以统一维护
- 后续重设计需要逐页替换硬编码样式

### 2.2 视觉方向割裂

主布局已经尝试了深色铁路控制台风格，但登录页、仪表盘、列表页、表单页仍大量保留 Ant Design 默认视觉。

影响：

- 系统没有统一品牌感
- 页面之间像不同阶段拼接产物
- “版本火车”业务隐喻没有形成稳定体验

### 2.3 页面层级重复

仪表盘内部存在独立 `Layout/Header/Content`，与外层 `MainLayout` 重复。

影响：

- 页面看起来像“布局套布局”
- 顶部栏语义不清
- 主内容区域间距和背景不一致

### 2.4 状态色不统一

系统中存在多套蓝色和多套状态色定义。

影响：

- 用户难以稳定理解颜色语义
- 状态标签、日历节点、统计卡片的颜色规则不一致
- 视觉噪音偏高

## 3. 设计目标

### 3.1 体验目标

- 让系统呈现为一个统一的企业级白底工作台
- 保留后台系统的高信息密度
- 强化版本火车的“班次、节点、状态、容量”业务特征
- 降低默认 Ant Design demo 感
- 提高列表、表单、日历等高频页面的扫描效率

### 3.2 技术目标

- 建立统一 Ant Design 主题 token
- 建立全局样式入口
- 抽取少量稳定通用组件
- 减少核心页面中的硬编码颜色、圆角、阴影、间距
- 保持现有路由、接口、权限、状态机不变

---

## 4. 排除清单

本次不做：

- 不改后端接口
- 不改数据库结构
- 不改权限矩阵
- 不改需求状态机
- 不改版本火车业务流程
- 不引入 Tailwind、shadcn/ui 或大型新 UI 框架
- 不做移动端专项重构，仅保证小屏不明显崩版
- 不做复杂动画系统
- 不做营销页或大屏展示页
- 不重写所有业务组件逻辑

## 5. 整体视觉方向

### 5.1 关键词

- 白底
- 清爽
- 高密度
- 调度工作台
- 企业级
- 状态清晰
- 表格优先
- 低装饰

### 5.2 页面气质

系统应更像“日常使用的版本发布调度系统”，不是深色驾驶舱，也不是营销型产品官网。

推荐观感：

- 主要背景为白色和极浅灰
- 卡片靠边框和轻微阴影建立层级
- 蓝色只用于主行动、当前选中、链接、纳版节点
- 风险、预警、完成、冻结分别使用稳定状态色
- 表格、筛选、页头、操作按钮保持紧凑

## 6. 主题 Token 设计

### 6.1 色彩

| 语义 | 建议值 | 使用场景 |
|------|--------|----------|
| 页面背景 | `#f7f9fc` | 主内容背景 |
| 卡片背景 | `#ffffff` | 卡片、表格容器、弹窗 |
| 主文本 | `#172033` | 标题、正文 |
| 次级文本 | `#667085` | 描述、辅助信息 |
| 弱文本 | `#98a2b3` | 空状态说明、元信息 |
| 边框 | `#e4e7ec` | 卡片、表格、输入框 |
| 分割线 | `#eef1f5` | 表格行、区块分隔 |
| 主色 | `#2563eb` | 主按钮、当前菜单、链接 |
| 主色浅底 | `#eff6ff` | 菜单选中、表格 hover |
| 成功 | `#16a34a` | 已完成、已投产 |
| 预警 | `#f59e0b` | 待确认、临近节点 |
| 风险 | `#dc2626` | 拒绝、阻塞、失败 |
| 紫色 | `#7c3aed` | 已纳版、开发中 |
| 冻结灰 | `#64748b` | 冻结、取消、不可操作 |

### 6.2 圆角

| 类型 | 建议值 |
|------|--------|
| 小按钮、Tag | 6px |
| 输入框、选择器 | 6px |
| 卡片、表格容器 | 8px |
| 弹窗 | 10px |

### 6.3 阴影

| 类型 | 建议值 | 使用场景 |
|------|--------|----------|
| 默认 | `0 1px 2px rgba(16, 24, 40, 0.04)` | 普通卡片 |
| 悬浮 | `0 4px 12px rgba(16, 24, 40, 0.08)` | hover 卡片 |
| 顶栏 | `0 1px 0 #e4e7ec` | Header |

### 6.4 间距

| 类型 | 建议值 |
|------|--------|
| 页面外边距 | 20px |
| 页面区块间距 | 16px |
| 卡片内边距 | 16px / 20px |
| 表单项间距 | 16px |
| 按钮间距 | 8px |

## 7. 全局布局设计

### 7.1 主布局

将现有深色侧边栏改为白底或极浅灰白底。

布局结构：

- 左侧导航：固定宽度 220px
- 顶部栏：高度 52px
- 内容区：浅灰白背景
- 页面容器：不再统一包一层大白卡，避免卡片套卡片

### 7.2 左侧导航

设计规则：

- 背景：白色
- 右边界：浅灰边框
- 当前菜单：淡蓝底 + 左侧蓝色竖线
- 图标：默认灰色，选中蓝色
- Logo 区：保留“版本火车”名称，但弱化 emoji 图标

### 7.3 顶部栏

设计规则：

- 白底
- 仅显示面包屑、返回按钮、用户信息
- 不承载页面标题，页面标题放到页面内容区的统一 PageHeader

### 7.4 内容区

设计规则：

- 背景：`#f7f9fc`
- 页面自身决定是否使用卡片
- 列表页优先使用白色表格容器
- 仪表盘允许多卡片布局
- 表单页允许左右分栏

## 8. 通用组件设计

### 8.1 AppPageHeader

用途：统一页面标题、说明、主操作和次操作。

布局：

- 左侧：标题、描述、状态摘要
- 右侧：主按钮、次按钮

适用页面：

- 仪表盘
- 需求池
- 需求详情
- 版本火车
- 班次列表
- 月历视图

### 8.2 FilterBar

用途：统一筛选区。

设计规则：

- 白底
- 浅灰边框
- 8px 圆角
- 筛选项横向排列，允许换行
- 查询按钮放在筛选项后方
- 新增按钮放入 PageHeader 主操作区，不混在筛选项中

### 8.3 DataCard

用途：统一普通卡片容器。

设计规则：

- 白底
- 浅灰边框
- 轻阴影
- 标题区高度统一
- 避免卡片套卡片

### 8.4 MetricCard

用途：统一统计卡片。

设计规则：

- 白底
- 左侧或顶部小色条表达状态
- 数字突出
- 不使用大面积彩色背景
- hover 时轻微描边和阴影变化

### 8.5 StatusTag

用途：统一需求状态、子状态、优先级、班次状态。

设计规则：

- 使用统一状态色映射
- 文案使用业务中文，不直接暴露枚举值
- 重要状态加深文字权重
- 避免一个页面内混用 Ant Design 预设色和自定义色

## 9. 核心页面改造方案

### 9.1 登录页

现状：灰底 + 默认 Card + emoji 标题。

改造：

- 使用纯白/浅灰白背景
- 登录卡片改为更轻的白色面板
- 标题改为“版本火车需求管理系统”
- 增加一句短说明：“统一管理需求评审、纳版、班次与投产”
- 按钮使用统一主色
- 不使用强装饰背景

### 9.2 主布局

现状：深色侧边栏 + 内容区统一大白卡。

改造：

- 侧边栏改为白底导航
- 内容区移除统一大白卡
- 页面自己使用 PageHeader、FilterBar、DataCard
- 顶栏只保留导航辅助功能

### 9.3 仪表盘

现状：内部重复 Layout，统计卡片大面积彩色背景。

改造：

- 移除内部 Layout
- 使用 AppPageHeader 显示“工作台”
- 顶部放角色、系统筛选、刷新按钮
- 统计卡改为白卡 + 小色条
- 关键时间节点改为紧凑列表或时间线
- 我的待办改为任务列表，突出“去处理”主操作

### 9.4 需求池列表

现状：筛选栏与新增按钮混排，表格默认感较强。

改造：

- PageHeader 放“需求池”和“新增需求”
- FilterBar 单独承载系统、状态、关键词、查询、重置
- 表格放入白色 DataCard
- 状态、优先级统一 StatusTag
- 行 hover 使用浅蓝底
- 操作按钮保持链接按钮，但收敛颜色和间距

### 9.5 需求创建/编辑表单

现状：双栏表单已有雏形，但样式仍偏默认。

改造：

- 保留左表单 + 右预览结构
- 左侧按“基本信息、描述、依赖、提交”分区
- 右侧固定为需求摘要、依赖风险、状态提示
- 表单区域减少重卡片嵌套
- 关键必填项视觉更明确

### 9.6 需求详情

改造：

- PageHeader 显示需求编号、标题、状态、操作按钮
- 主体分为基础信息、描述、依赖、变更记录
- 状态和子状态使用统一 StatusTag
- 操作按钮按当前角色权限展示，视觉优先级明确

### 9.7 版本火车列表

现状：普通表格页面。

改造：

- PageHeader 放“版本火车”和创建火车按钮
- 表格容器统一为 DataCard
- 火车名称、班次数、容量、系统数做更强扫描层级
- 班次列表入口作为次操作

### 9.8 班次列表与班次详情

改造：

- 班次列表突出状态、纳版日、封板日、投产日
- 班次详情顶部展示班次摘要和关键日期
- 容量、需求池、纳版动作放入明确区块
- 智能纳版入口作为当前页面重点动作，但不做过度 AI 装饰

### 9.9 月历视图

现状：功能方向正确，但视觉和外层容器可统一。

改造：

- PageHeader 显示“月历视图”
- 筛选区使用 FilterBar
- 月历卡片保持白底
- 班次条继续使用彩色条，但颜色进入统一 token
- 里程碑图例固定为纳版蓝、封板橙、投产绿
- 底部班次摘要卡片改为统一 DataCard 样式

## 12. 风险与控制

### 12.1 风险：inline style 分散导致改造范围扩大

控制：

- 先做主题和通用组件
- 优先改核心页面
- 非核心细节逐步替换，不一次性全量重写

### 12.2 风险：页面结构调整影响交互

控制：

- 不改变业务函数和接口调用
- 页面改造以渲染结构和样式为主
- 每改一个页面立即运行对应验证

### 12.3 风险：Ant Design 默认样式覆盖困难

控制：

- 优先使用 ConfigProvider token
- 必要时使用统一 CSS class 覆盖
- 避免大量 `!important`
