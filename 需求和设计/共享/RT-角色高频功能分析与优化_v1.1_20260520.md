# 角色高频功能分析与优化建议

**版本号**: v1.1
**日期**: 2026年5月20日
**视角**: 敏捷教练
**目标**: 从每个角色的日常工作场景出发，提供落地的优化建议

***

## 一、角色高频场景总览

| 角色    | 高频功能Top5                          | 最痛苦的场景             |
| ----- | --------------------------------- | ------------------ |
| 业务BA  | 查看自己系统的需求、录入需求、发起评审、跟进需求进度、子状态变更  | 每次都要手动筛选自己系统的需求    |
| 产品经理  | 查看全系统需求、评审需求、协调班次规划、跟进重点需求        | 需要频繁切换火车查看不同班次     |
| 项目经理  | 监控整体进度、子状态变更审核、协调跨角色问题、风险管理、处理阻塞点 | 找不到项目整体视图，看不到跨系统风险 |
| 火车管理员 | 管理火车和班次、纳版需求、状态变更、监控容量、投产确认       | 纳版时需要反复核对容量和依赖     |
| 技术经理  | 查看自己系统的需求、跟踪开发进度、协调开发资源           | 需要快速找到自己负责系统的需求    |
| 测试负责人 | 跟进测试进度、确认投产需求、处理紧急变更              | 找封板后需要测试的需求慢       |

***

## 二、角色深度分析与优化建议

### R2.1 业务BA优化

#### 角色画像

- **日常工作**：每天录入几个需求，定期查看自己系统的需求状态，跟进需求从草稿到投产的全流程
- **痛点**：每次都要筛选「自己系统」，找最近的班次慢，看不到需求的整体进度
- **优先级**：P0

#### 高频场景分析

| 场景             | 频率   | 当前流程                     | 问题       |
| -------------- | ---- | ------------------------ | -------- |
| 登录后查看需求        | 每天多次 | 1.进入需求列表 2.筛选「归属系统」=自己系统 | 每次都要手动选  |

| 录入需求           | 每天几次 | 正常表单                     | 表单默认值可优化 |
| 跟进需求评审状态       | 每周多次 | 进入详情页查看                  | 可以优化     |

**OP-BA-02 需求列表默认筛选优化**

- 需求列表页自动筛选：「归属系统」= 自己负责的系统
- 需求列表页自动筛选：「状态」≠ 已取消、已投产（默认只看活跃需求）
- 需求列表页显示「上次筛选」快捷按钮（如「我的活跃需求」）

**OP-BA-03 录入需求表单默认值优化**

- 归属系统：默认选择自己负责的系统
- 业务归属人：默认选择自己
- 创建需求后自动跳转到「发起评审」页面（可选跳过）

**OP-BA-04 纳版时间提醒**
- 在需求详情页和列表页显示纳版截止日期
- 在班次纳版截止前3天，BA首页显示「⚠️ 您的需求即将纳版」提醒
- 支持设置个人提醒（如提前3天/7天邮件通知）
