# 需求变更确认智能体设计方案

## 一、智能体概述

### 1.1 定位
一个专门用于产品需求沟通场景的智能助手，能够实时分析对话内容，自动识别需求变更，生成结构化的变更摘要，并提供确认消息和风险提醒。

### 1.2 核心能力
- ✅ **自动检测变更**：识别对话中提到的需求变更点
- ✅ **历史对比**：与之前的需求版本进行对比
- ✅ **变更影响分析**：分析变更对项目进度、成本、资源的影响
- ✅ **风险提醒**：识别潜在风险并给出提醒

### 1.3 输出形式
- 📋 结构化摘要（变更详情表格）
- 💬 确认消息（可直接发送的确认文本）
- 📝 变更日志（历史变更记录）

## 三、核心逻辑设计

### 3.1 变更检测规则

#### 变更类型识别
| 变更类型 | 关键词/模式 | 示例 |
|---------|------------|------|
| 新增需求 | "增加"、"添加"、"新功能"、"还需要" | "还需要加一个导出功能" |
| 需求修改 | "改成"、"调整为"、"修改为"、"优化" | "把按钮改成红色" |
| 需求删除 | "去掉"、"删除"、"不需要"、"取消" | "这个模块先不做" |
| 范围变更 | "范围"、"包含"、"不包含" | "范围扩大到全部用户" |
| 时间变更 | "延期"、"提前"、"截止日期"、"上线时间" | "上线时间推迟一周" |
| 优先级变更 | "优先级"、"紧急"、"重要"、"先" | "这个优先级调到最高" |

#### 变更要素提取
```
变更要素 = {
  需求ID/名称: string,
  变更类型: "新增" | "修改" | "删除",
  变更前: string,
  变更后: string,
  提出人: string,
  提出时间: datetime,
  变更原因: string,
  涉及模块: string[]
}
```

### 3.2 影响分析维度

| 维度 | 分析内容 | 评估标准 |
|-----|---------|---------|
| 工作量 | 新增/减少的工作量 | 人天/人时 |
| 进度 | 对里程碑的影响 | 延期天数 |
| 依赖 | 受影响的上下游需求 | 依赖清单 |
| 成本 | 预算影响 | 金额变化 |
| 质量 | 对现有功能的影响 | 风险等级 |

### 3.3 风险识别清单

| 风险类型 | 识别信号 | 风险等级 |
|---------|---------|---------|
| 技术风险 | 技术难点、新技术、架构改动 | 高/中/低 |
| 进度风险 | 时间压缩、并行任务多 | 高/中/低 |
| 资源风险 | 人员变动、技能不足 | 高/中/低 |
| 沟通风险 | 多方参与、意见不一致 | 高/中/低 |
| 范围蔓延 | 频繁变更、需求膨胀 | 高/中/低 |

## 五、智能体提示词设计

### 5.1 系统提示词（System Prompt）

```
你是一位专业的产品需求变更管理助手，擅长在对话中识别、分析和确认需求变更。

## 你的职责
1. 实时分析对话内容，识别需求变更点
2. 对比历史需求，明确变更前后的差异
3. 评估变更对项目的影响（工作量、进度、成本）
4. 识别潜在风险并给出提醒
5. 生成结构化的变更摘要和确认消息

## 变更类型定义
- 新增：对话中提到要增加之前没有的功能或需求
- 修改：对已有需求的调整、优化、变更
- 删除：取消或去掉之前计划的功能或需求
- 范围变更：需求边界的扩大或缩小
- 时间变更：交付时间、截止日期的调整
- 优先级变更：需求重要程度、紧急程度的调整

## 输出要求
1. 结构化摘要：使用表格展示变更详情，包含变更前后对比
2. 确认消息：语气专业、友好，便于直接发送给相关方
3. 变更日志：按时间倒序排列，便于追溯历史

## 分析维度
- 工作量影响（人天/人时）
- 进度影响（延期天数）
- 成本影响（金额变化）
- 技术风险（高/中/低）
- 依赖影响（受影响的上下游）

## 注意事项
- 如果对话中没有明确的变更，输出"未检测到需求变更"
- 变更影响评估基于合理推测，使用"预计"、"可能"等措辞
- 风险提醒要具体，说明风险原因和应对建议
```

### 5.2 用户提示词模板（User Prompt）

```
请分析以下对话内容，识别需求变更并生成相应输出：

## 历史需求（如有）
{history_requirements}

## 当前对话
{conversation_content}

## 输出要求
请提供以下三部分内容：

1. 📋 结构化摘要：使用表格展示所有变更项，包含变更类型、变更前后对比、影响评估

2. 💬 确认消息：生成一段可直接发送给需求提出方的确认消息，包含：
   - 变更内容总结
   - 影响说明（工作量、进度等）
   - 风险提示
   - 确认请求

3. 📝 变更日志：按时间顺序记录本次变更，格式参考历史日志

如果对话中没有检测到需求变更，请明确说明。
```

## 六、使用示例

### 示例1：产品开发场景

**输入对话：**
```
产品经理：咱们这个报表功能，业务方说还需要加一个导出Excel的功能
开发负责人：这个之前没排进去，现在加的话可能要延期
产品经理：延期多久？
开发负责人：大概2天吧，还要做性能测试
产品经理：那按钮颜色他们说要改成红色，突出一点
开发负责人：这个简单，半天搞定
产品经理：对了，短信通知功能他们说先不做了，预算不够
开发负责人：那正好，省1天工作量
```

**智能体输出：**

📋 **结构化摘要**

| 序号 | 需求项 | 变更类型 | 变更前 | 变更后 | 影响评估 |
|-----|-------|---------|-------|-------|---------|
| 1 | Excel导出 | 新增 | 无 | 支持Excel导出 | 工作量+2人天，需性能测试 |
| 2 | 按钮颜色 | 修改 | 默认蓝色 | 红色（突出显示） | 工作量+0.5人天 |
| 3 | 短信通知 | 删除 | 计划实现 | 取消 | 工作量-1人天 |

**总体影响：**
- 净工作量变化：+1.5人天
- 进度影响：预计延期1-2天
- 新增风险：导出功能性能风险

💬 **确认消息**

根据沟通，本次需求变更如下：
1. 新增Excel导出功能（+2人天，需性能测试）
2. 按钮颜色改为红色（+0.5人天）
3. 取消短信通知功能（-1人天）

整体工作量增加1.5人天，预计延期1-2天。

⚠️ 风险提示：导出功能可能影响系统性能，建议增加专项测试。

请确认以上变更是否准确？

## 九、总结

本智能体设计方案提供了一个完整的需求变更确认解决方案，具备以下特点：

1. **智能化**：自动识别对话中的变更点，无需人工标记
2. **全面性**：覆盖变更检测、影响分析、风险提醒全流程
3. **实用性**：输出格式可直接用于工作场景
4. **可扩展**：架构清晰，便于后续功能扩展

通过部署此智能体，可以有效提升需求变更管理的效率和准确性，减少沟通成本和变更风险。
