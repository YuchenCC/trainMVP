# 需求变更智能体 - 分步设计方案

## 整体流程概览

```
用户对话@智能体
    ↓
提取关键信息（系统名+需求概述）
    ↓
调接口查询版本火车系统
    ↓
获取存量需求详情（状态、依赖、负责人等）
    ↓
智能分析变更内容
    ↓
生成变更对比和影响分析
    ↓
输出结构化结果 + 同步系统
```

---

## 第一步：对话触发与参数提取

### 1.1 用户输入格式设计

```
@需求变更助手

【系统】订单管理系统
【需求概述】
业务方：订单导出功能需要支持Excel格式，目前只支持CSV
开发：这个要改导出模块，预计增加2人天
业务方：那6月15日能上线吗？
开发：时间有点紧，可能要延期到6月20日
```

### 1.2 CoZe 智能体 - 参数提取节点

**节点名称**：提取关键信息

**输入**：用户原始消息

**输出**：结构化参数

```json
{
  "system_name": "订单管理系统",
  "requirement_summary": "订单导出功能支持Excel格式，延期到6月20日",
  "conversation": "业务方：订单导出功能需要支持Excel格式...",
  "extracted_keywords": ["导出", "Excel", "CSV", "延期"]
}
```

**提示词**：
```
请从用户输入中提取以下信息：

1. 系统名称：标记为【系统】或【系统名】后的内容
2. 需求概述：【需求概述】后的内容，或整段对话的核心主题
3. 完整对话：用户提供的所有对话内容

输出JSON格式：
{
  "system_name": "系统名称",
  "requirement_summary": "一句话概括需求变更",
  "conversation": "完整对话内容",
  "extracted_keywords": ["关键词1", "关键词2"]
}

如果缺少系统名称，输出：{"error": "缺少系统名称，请使用【系统】XXX 格式提供"}
```

```
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 匹配存量需求                                         │
└─────────────────────────────────────────────────────────────┘

  - 系统名：订单管理系统
  - 关键词：["导出", "Excel", "CSV"]
  - 需求概述：订单导出功能支持Excel格式

  1. 精确匹配：需求名称包含"导出"
```

## 第五步：CoZe 完整工作流配置

### 5.1 工作流节点图

```
[开始]
  ↓
[节点1: 提取参数]
  - 输入: 用户消息
  - 输出: system_name, requirement_summary, conversation
  - 判断: 参数是否完整?
    ├─ 否 → [提示补充信息] → [结束]
    └─ 是 → 继续
  ↓
[节点2: 查询需求列表]
  - 调用: search_requirements
  - 参数: system_name, keywords
  - 输出: requirements[]
  ↓
[节点3: 匹配需求]
  - 逻辑: 根据关键词和描述匹配
  - 输出: matched_requirement
  - 判断: 是否匹配到?
    ├─ 否 → [提示未找到相关需求] → [结束]
    └─ 是 → 继续
  ↓
[节点4: 获取需求详情]
  - 调用: get_requirement_details
  - 参数: requirement_id
  - 输出: 需求详情 + 依赖关系
  ↓
[节点5: 变更分析]
  - 输入: 对话内容 + 存量需求详情
  - 处理: LLM分析变更点
  - 输出: 结构化分析结果
  ↓
[节点6: 生成输出]
  - 生成: 分析报告 + 确认消息 + 变更日志
  ↓
[节点7: 同步飞书]
  - 发送: 个人确认消息
  - 推送: 群聊摘要卡片
  ↓
[节点8: 同步版本火车]
  - 调用: create_change_request
  - 参数: 分析结果
  ↓
[节点9: 返回结果]
  - 输出: 处理完成汇总
  ↓
[结束]
```

### 5.2 异常处理分支

```
异常场景处理:

1. 未找到系统
   → 提示: "未找到系统'{system_name}'，请检查系统名称是否正确"
   
2. 未匹配到需求
   → 提示: "未在系统'{system_name}'中找到相关需求，是否创建新需求？"
   
3. API调用失败
   → 提示: "系统查询失败，请稍后重试或联系管理员"
   → 记录错误日志
   
4. 未检测到变更
   → 提示: "未从对话中检测到明确的变更点，请补充变更详情"
```
