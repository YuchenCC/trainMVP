## 一、测试范围

| 范围 | 是否纳入 | 说明 |
|------|----------|------|
| 全局班次列表查询 | 是 | `GET /api/schedules`，进入 `/schedules` 默认加载 |
| 按火车切换班次列表 | 是 | 选择“所属火车”后调用 `GET /api/trains/:trainId/schedules` |
| 班次列表分页 | 是 | 全局班次列表支持分页；按火车查询当前不分页 |
| 班次列表字段展示 | 是 | 按当前代码字段验证，并记录“所属火车”列未展示差距 |
| 班次详情跳转 | 是 | 点击表格行进入详情页 |
| 新增班次入口 | 是 | 必须先选择所属火车 |
| 编辑班次入口 | 是 | 点击编辑打开弹窗 |
| 班次状态操作 | 是 | 开始、封板、投产、取消 |
| 角色权限按钮显隐 | 部分纳入 | 后端有权限，前端按钮当前未完整按角色隐藏，作为差距验证 |
| 默认显示当前用户相关火车 | 暂不纳入 | 本轮确认先不做用户相关火车过滤和默认选中 |

### TC2.3-API-01 查询全局班次列表

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证登录用户可以查询全局班次列表 |
| **前置条件** | 用户已登录；系统中存在多个班次 |

| **预期结果** | 1. 返回 HTTP 200<br>2. `success=true`<br>3. 返回 `list` 和 `pagination`<br>4. `list` 中包含班次 ID、火车 ID、火车名称、班次名称、状态、关键日期、容量和需求数 |

## 五、差距验证测试案例

> 本节仅保留本轮确认需要跟踪的差距项。“用户相关火车过滤”和“默认选中相关火车”暂不纳入本轮测试范围。

### TC2.3-GAP-01 班次列表未展示所属火车列

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证班次列表字段是否包含所属火车 |
| **前置条件** | 用户已登录；存在多个火车下的班次 |
| **测试步骤** | 1. 打开 `/schedules`<br>2. 查看表格列 |
| **当前结果** | 后端返回 `trainName`，但前端表格未展示“所属火车”列 |
| **期望结果** | 表格展示“所属火车”列 |
| **结论** | 当前为差距项 |
| **优先级** | P1 |

### TC2.3-GAP-02 按火车查询班次不支持后端分页

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证选择某辆火车后是否仍支持后端分页 |
| **前置条件** | 火车 A 下存在超过 20 条班次 |
| **测试步骤** | 1. 打开 `/schedules`<br>2. 选择火车 A<br>3. 观察分页行为或调用 `GET /api/trains/{trainAId}/schedules?page=2&pageSize=20` |
| **当前结果** | 指定火车班次接口当前返回全部班次，不处理分页参数，不返回 `pagination` |
| **期望结果** | 指定火车班次接口支持分页，并返回 `pagination` |
| **结论** | 当前为差距项 |
| **优先级** | P1 |

### TC2.3-GAP-03 前端操作按钮未按角色隐藏

| 项目 | 内容 |
|------|------|
| **测试目的** | 验证班次列表操作按钮是否根据当前用户角色动态显示 |
| **前置条件** | BA 或普通用户登录；存在计划中、进行中或封板班次 |
| **测试步骤** | 1. 打开 `/schedules`<br>2. 查看班次操作列 |
| **当前结果** | 前端仍可能展示编辑、取消、开始、封板、投产等管理类按钮；后端接口会做权限拦截 |
| **期望结果** | 非火车管理员只看到允许的查看类操作；管理类按钮前端隐藏 |
