## 一、需求概述

### 1.1 背景

当前需求提交评审前缺乏前置自动质量检测环节，大量存在格式错误、描述模糊、缺失验收条件等问题的需求直接进入评审流程，导致人工评审耗时久、返工率高。需要引入 AI 智能审查能力，在需求提交前自动完成质量检查，从源头提升需求质量。

### 1.2 目标

- 在需求创建/编辑页面提供「AI 审查」按钮，提交评审前可自动检查需求质量
- 审查覆盖：用户故事格式、描述完整性、验收条件、优先级合理性、歧义表述等维度
- 审查完成后给出质量评分（0-100分）和可落地的改进建议
- AI 审查不可用时自动降级到本地规则审查，不影响核心流程

### 1.3 核心价值

| 指标 | 现状 | 目标 |
|------|------|------|
| 需求评审返工率 | ~42% | 降低至 25% 以下 |
| 单需求平均评审耗时 | 30+ 分钟 | 缩短至 20 分钟以下 |
| 需求格式规范率 | 未统计 | 90%+ 符合用户故事格式 |
| 验收条件覆盖率 | 未统计 | 100% 需求包含验收条件 |

### 1.4 用户故事

作为业务BA，我想要在需求创建页面点击「AI审查」按钮，以便在提交评审前自动检查需求描述是否完整规范，确保我提交的需求不会因为格式错误或描述不清被驳回返工。

### 1.5 排除清单

- 不实现主动触发（如自动保存时自动审查），仅支持手动点击触发
- 不存储审查记录到独立表（使用需求操作日志记录审查行为即可）
- 不实现审查报告导出 PDF
- 不实现审查结果的多版本对比
- 不实现自定义审查规则配置（本期使用固定规则）

## 二、业务规则

### 2.1 审查维度

| 维度 | 检查项 | 触发条件 | 严重程度 |
|------|-------|---------|---------|
| 标题规范 | 标题长度 ≥ 5 字符 | 标题 < 5 字符 | medium |
| 描述完整性 | 描述长度 ≥ 50 字符 | 描述 < 50 字符 | high |
| 用户故事格式 | 匹配"作为...我想要...以便..." | 不匹配 | medium |
| 优先级设置 | 必须在 P0/P1/P2/P3 中 | 未设置或无效 | high |
| 工作量评估 | 故事点数 1-100 | 不在范围内 | medium |
| 系统归属 | 必须关联系统 | 未关联 | high |
| BA 分配 | 必须指定 BA | 未分配 | high |
| 验收条件 | AI 检测是否隐含验收条件 | — | AI 评估 |
| 歧义表述 | AI 检测模糊、歧义描述 | — | AI 评估 |
| 业务价值 | AI 检测是否描述业务价值 | — | AI 评估 |

### 2.2 评分规则

- 本地规则问题按严重程度扣分：high 扣 20 分、medium 扣 10 分、low 扣 5 分
- AI 评分由 Coze 智能体独立评估，返回 0-100 分
- 最终评分 = 本地规则得分 × 70% + AI 得分 × 30%
- 及格线：60 分

### 2.3 状态流转

```
需求表单必填项 → 点击「AI审查」 → AI审查中 → 审查结果展示 → 用户关闭 / 修改后重新审查
                                     ↓（失败/超时）
                                降级到本地规则审查
```

## 三、状态机

```
用户填写需求表单
       │
       ├── 点击「AI 审查」按钮
       │       │
       │       ├── 必填项校验不通过 → Toast 提示"请先填写必填项"
       │       │
       │       └── 校验通过 → 弹出审查 Modal（loading 动画）
       │               │
       │               ├── AI 审查成功 → 展示审查报告（分数 + 问题Tab + 建议Tab + 验收条件Tab）
       │               │       │
       │               │       └── 用户点击「我知道了」关闭 Modal
       │               │
       │               └── AI 审查失败/超时 → 降级到本地规则审查 → 展示审查报告
       │                       │
       │                       └── 用户点击「我知道了」关闭 Modal
       │
       └── 用户手动修改表单 → 可再次点击「AI 审查」重新审查
```

## 五、用户界面

### 5.1 审查触发

在需求创建/编辑表单的顶部操作栏，提供「AI 审查」按钮（RobotOutlined 图标），与保存、提交按钮并列。

### 5.2 审查中状态

弹出 Modal，展示 loading 动画，包含 4 步检查进度提示：
1. 标题检查 — 检查标题是否简洁清晰
2. 用户故事格式 — 验证描述是否符合标准格式
3. 验收条件 — 检查是否包含明确的验收标准
4. 优先级与工作量 — 评估优先级和故事点是否合理

### 5.3 审查结果展示（Tabs 布局）

- **评分头部：** 渐变色卡片 + 圆形评分 badge，按通过（≥60）/ 需改进（40-59）/ 问题较多（<40）三种状态显示不同颜色
- **AI 优化建议：** 紫色卡片，展示 AI 优化后的标题和完整描述
- **⚠️ 问题 Tab：** 按严重程度（严重/中等/轻微）分组展示，每项包含问题描述 + 改进建议
- **💡 建议 Tab：** 蓝色编号列表
- **✅ 验收条件 Tab：** AI 建议的验收条件，绿色勾号列表

### 5.4 操作

- 复制按钮：合并复制优化建议 + 验收条件
- 「我知道了」按钮：关闭 Modal
