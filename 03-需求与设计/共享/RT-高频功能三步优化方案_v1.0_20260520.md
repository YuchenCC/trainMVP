## 二、优化方案详情

### OP2.1 创建班次优化（P0）

#### 问题
- 当前5-6步，超高频功能
- 缺少关键日期预览，用户可能需要反复修改
- 缺少容量概览，无法提前评估

#### 优化后流程（3步）
```
1. 进入班次列表页，选择所属火车
2. 选择开始/结束日期 → 自动显示预览关键日期和容量概览
3. 点击「创建」按钮
```

### OP2.2 创建需求优化（P1）

#### 问题
- 当前5-6步，超高频功能
- 表单字段过多
- 依赖需求操作复杂

#### 优化后流程（3步）
```
1. 进入需求列表页，点击「新增需求」
2. 快速填写核心字段（标题、归属系统、优先级、业务归属人）
3. 点击「保存」，其他字段后续补充

### OP2.3 子状态变更优化（P1）

#### 问题
- 当前4步，高频功能
- 需要进入详情页，打开弹窗，选择，确认

#### 优化方案1：列表页快捷操作（2步）
```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│  需求列表                                                                [搜索]    │
├─────────────────────────────────────────────────────────────────────────────────────┤
│  REQ-001  用户登录优化  [已纳版-开发中]  [▼] [编辑] [发起评审] [取消]  │
│               │                                              │
│  ┌─────────────────────────────────────┐                      │
│  │ 子状态变更（快捷）             │                      │
│  ├─────────────────────────────────────┤                      │
│  │ • 开发中（当前）                   │                      │
│  │ ➤ SIT测试                          │                      │
│  │   UAT测试                         │                      │
│  │   封板                            │                      │
│  └─────────────────────────────────────┘                      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

#### 优化方案2：详情页优化（2步）
```
┌─────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────┤

#### 验收条件

- [ ] AC-OP2.3.3: 子状态变更不超过2步

### OP2.4 从火车移除需求优化（P2）

#### 问题
- 当前4步，中频功能
- 必须填写原因，可能繁琐

#### 优化方案：双模式
```
┌─────────────────────────────────────────────────────────────────┐
│  从班次移除需求                                                  │
├─────────────────────────────────────────────────────────────────┤
│  确定要移除需求「REQ-001 用户登录优化」吗？                        │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ ⏱️ 快捷移除（无需原因）：适用于误操作、需求拆分等场景            ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 📝 标准移除（必填原因）：适用于需求取消等需要留痕场景         ││
│  │ 移除原因 *：[________________________________________]        ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                  │
│                          [取消]  [快捷移除]  [标准移除]          │
└─────────────────────────────────────────────────────────────────┘
```

#### 验收条件
- [ ] AC-OP2.4.1: 提供快捷移除模式，无需填写原因（2步）
- [ ] AC-OP2.4.2: 提供标准移除模式，必填原因（3步）
- [ ] AC-OP2.4.3: 快捷移除记录默认原因，如「需求拆分」

### OP2.5 紧急变更申请优化（P2）

#### 问题
- 当前4步，低频但紧急场景需要快速操作

#### 优化方案：双模式
```
┌─────────────────────────────────────────────────────────────────┐
│  紧急变更申请                                                  │
├─────────────────────────────────────────────────────────────────┤
│  ⏱️ 快速紧急变更（P1默认）                                        │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 变更原因 *：[________________________________________]      ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                  │
│  📝 完整紧急变更（P0需要）                                        │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 紧急程度：[P0 ▼] [P1]                                      ││
│  │ 变更原因 *：[________________________________________]      ││
│  └─────────────────────────────────────────────────────────────┘│
│                                                                  │
│                          [取消]  [快速提交]  [完整提交]          │
└─────────────────────────────────────────────────────────────────┘
```

#### 验收条件
- [ ] AC-OP2.5.1: 快速模式默认P1，仅需填写原因（2步）
- [ ] AC-OP2.5.2: 完整模式可选P0/P1，填写完整信息（3步）
- [ ] AC-OP2.5.3: 快速模式提交后记录默认紧急程度P1
