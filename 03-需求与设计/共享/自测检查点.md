# 自测检查点清单

> 本清单用于开发自测阶段的全面检查，涵盖交互类、管理类、交易类、文件相关、复杂逻辑、配置类、数据库、定时任务、界面类、安全等各大类测试点。

---

## 一、交互类

### 1. 接口

- [ ] 前后端校验规则是否一致，字段关联、校验规则（必填/格式/范围）（如前端允许 11 位手机号，后端仅允许 10 位）
- [ ] 调用接口需配置合理的超时时间
- [ ] 报文格式/长度/加密规则
- [ ] 防重发，如 Nonce、Timestamp、签名验证加频率限制
- [ ] 调用频率限制，限流/熔断/开关等
- [ ] 正常流程测试：确保接口在标准输入下返回正确的输出
- [ ] 边界条件测试：测试接口在参数取最大值、最小值、空值、特殊字符等极限情况下的表现
- [ ] 错误处理测试：与提供方系统确认接口返回的特殊异常码（包括查无记录等），验证接口返回的各种错误码，有做出相应的正确处理逻辑
- [ ] 重试机制测试：验证接口是否能够正确重试

### 2. 交互

- [ ] 提交/确认按钮添加 loading，防止重复点击
- [ ] 接口异常时仅返回业务必要的报错信息，并针对各类报错做友好的异常处理
- [ ] 敏感操作二次确认，如确认是否删除

---

## 二、管理类功能

### 1. 权限控制

- [ ] 用户、角色、权限控制符合业务需求

### 2. 数据操作

- [ ] 查询语句含分页、且索引生效的执行计划自测用例，分页逻辑正常（如最后一页翻页空数据、页码越界报错）
- [ ] 涉及表数据的任何操作（增删改查），请求内容需与表数据进行准确性核对，涉及批量插入/更新的，还需按照生产数据总量验证处理耗时
- [ ] 重要参数校验（比如涉及是否能命中索引的字段、模糊查询未转译特殊字符等），避免一次性加载过多数据

### 3. 批量操作

- [ ] 是否涉及复杂业务逻辑不拆分、长事务导致数据库死锁
- [ ] 打印大量日志或者批量生成临时文件导致资源不足
- [ ] 数据不一致：1、部分提交或回滚不彻底；2、幻读/不可重复读；3、异步任务导致（如同步 ES、缓存更新策略等）

### 4. 审计

- [ ] 对数据操作是否满足审计要求

---

## 三、交易类功能

### 1. 数据一致性

- [ ] 端到端链路测试，模拟完成交易流程，检查数据库变更情况、检查核心系统数据最终是否一致，并截图

### 2. 金额类型

- [ ] 单笔、批量、手续费等计算是否正确
- [ ] 金额精度、金额格式验证，是否存在金额溢出或数据类型错误、最大最小值限制、负数、零值、非法字符等

### 3. 账务处理验证

- [ ] 借贷方向验证：每笔交易的借方与贷方是否正确
- [ ] 是否遵循"有借必有贷，借贷必相等"原则
- [ ] 是否正确区分内部账户与外部账户的借贷方向
- [ ] 确认账务分录验证、余额更新、流水记录验证等
- [ ] 账务冲正、对账验证，是否支持交易冲正（冲销、撤销等）、是否与外部系统（如银联、支付平台、第三方对账系统）进行对账，结果是否正确

### 4. 状态流转

- [ ] 验证所有执行状态、审批状态、交易状态（如"待支付"→"已支付"→"已退款"）的流转是否符合业务规则
- [ ] 确保无效状态跳转（如"已退款"直接转为"已发货"）被拦截
- [ ] 记账交易是否正确记录事务处理的中间状态，并根据关联事务的状态进行提交确认或回滚

### 5. 异常回滚/补偿机制

- [ ] 异常场景模拟测试：验证是否触发回滚（如取消订单、释放库存）或补偿操作（如自动退款）
- [ ] 服务中断恢复测试：重启核心服务后，检查未完成的事务是否通过补偿机制恢复，补偿操作是否幂等

### 6. 幂等设计

- [ ] 重复请求测试：使用相同参数（如订单号、金额）重复提交关键请求（如支付、退款）。验证系统是否仅处理第一次请求，后续请求返回"已处理"或"操作冲突"
- [ ] 幂等标识有效期测试：设置幂等标识的存储时间（如 Redis 的 EXPIRE），模拟超时后重复请求是否被允许

### 7. 序列号/流水号

- [ ] 生成规则是否合理，使用合适成熟的序号生成器算法，比如高低位算法、雪花算法等，根据数据量选择合适的序号长度，避免出现序号不足或长度溢出；分表分库不使用数据库自增机制
- [ ] 唯一性保障（分布式防重复）：高并发压力测试、分布式节点测试
- [ ] 如涉及上下游，还需考虑跨系统格式兼容

### 8. 审计

- [ ] 交易日志审计，日志要素、存储时间等

## 四、文件相关

### 1. 文件上传

- [ ] 仅允许上传特定类型文件，且不能通过文件后缀进行判断，需检查文件头中的文件类型
- [ ] 对上传的文件大小进行限制，一般不得超过 10MB
- [ ] 上传文件时防止重复提交上传申请

### 2. 文件存储

- [ ] 同路径下文件数量根据系统确定，不宜过多
- [ ] 文件名防重复设计，避免覆盖

### 3. 资源释放

- [ ] 使用 IO 流写文件完成后，是否及时关闭，避免在高并发情况下造成后续写文件功能异常
- [ ] 及时清理临时目录和文件或者使用完删除，需要根据文件数量评估清理频率

### 4. 文件下载

- [ ] 不得通过文件的相对路径进行下载，防止通过修改参数获取服务器上文件
- [ ] 导出时需限制最大记录数
- [ ] 批量下载没有并发控制或限速控制，导致系统资源耗尽

## 五、复杂逻辑

### 1. 代码

#### 循环与递归

- [ ] 覆盖循环次数的边界值用例（如 0 次、最大次数、数据总量）
- [ ] 递归深度的极限范围用例

#### 多线程与异步

- [ ] 多线程场景下的线程安全性和数据一致性自测用例
- [ ] 高并发场景下异步逻辑的稳定性用例（结合性能测试）

#### 复杂算法（排序、搜索、动态规划等）

- [ ] 设计不同规模的测试数据（如小数据、大数据）用例
- [ ] 算法的输出是否符合预期，性能是否满足要求的自测用例

---

## 六、配置类

### 1. 配置文件

- [ ] 配置参数对应功能修改前、后的结果差异对比
- [ ] 新增配置文件参数必须附默认值检查，并确认研发测试环境与生产的一致性用例

### 2. 环境变量

- [ ] 变量被正确读取和应用的截图

### 3. 应用参数

- [ ] 开关类功能按预期效果生效的用例
- [ ] 参数设计边界值自测用例（包括但不限于最大值、最小值、空值）

### 4. 脚本

- [ ] 生产检查脚本执行的文件权限、空间、目录等

---

## 七、数据库

### 1. 库表设计

- [ ] 字段类型/长度/默认值、主键/外键约束、唯一索引
- [ ] 索引缺失或冗余
- [ ] 存量数据变更，需检查生产数据是否需要初始化、满足变更后的要求
- [ ] 数据表变更对上游、下游无影响自测用例

### 2. 数据迁移/重构

- [ ] 具有备份原始数据、且可正常恢复使用的自测用例
- [ ] 生产数据总量执行的耗时时长用例

### 3. 供数

- [ ] 按照定时任务配置的命令进行试推成功用例
- [ ] 异常数据是否通过数据核验
- [ ] 按照定时任务配置的到点供数的用例

### 4. 订阅

- [ ] 生产数据总量订阅解析入库用例，订阅执行失败要确保上日数据的可用性
- [ ] 对应业务功能可正确使用订阅数据的自测用例

---

## 八、定时任务

### 1. 调度设计

- [ ] 校验 Cron 表达式有效性（如跨月/跨年场景），检查任务执行时间准确性

### 2. 容错机制

- [ ] 重试次数/间隔、重试终止条件、失败告警、管台手动重跑等

### 3. 异常控制

- [ ] 开关是否可以停止正在执行中的定时任务
- [ ] 异常数据是否标记跳过，避免异常数据堆积

### 4. 并发控制

- [ ] 双集群、或多活是否有多节点分布式锁控制，避免重复执行
- [ ] 幂等设计（是否可以重复执行，数据不会重复处理）

---

## 九、界面类

### 1. 向下兼容

- [ ] 含新版本能正确处理旧版本的数据和功能、旧版本升级到新版本后的兼容

### 2. 环境兼容

- [ ] 测试不同操作系统（Windows、Linux、macOS）和浏览器版本下的兼容性、屏幕尺寸及分辨率的兼容性测试

---

## 十、安全

### 1. 输入输出

- [ ] 分页参数是否设置合理最大值，避免不合理的深度分页或单页数据过多的情况发生，（如果参数配置不能由业务人员修改）
- [ ] 提供给前端或移动端使用的根据资源标识查询资源信息的 API，资源标识是否具备不可猜测性
- [ ] XSS 攻击、SQL 注入风险

### 2. 访问控制

- [ ] 短信炸弹：限制 ip、手机号操作频率
- [ ] 是否存在越权风险，主要是 API，操作者与被操作资源之间的权限判断应完整闭合。水平越权和垂直越权

### 3. 数据安全

- [ ] 新增打印日志是否对敏感信息进行脱敏处理
- [ ] 出入参是否包含高等级数据或敏感数据，需进行字段级加密
- [ ] 响应数据是否存在暴露内部资源地址、内部网络信息等风险
- [ ] 面客功能展示敏感信息时，是否根据行内相关要求进行脱敏，且脱敏行为不是前端进行的

### 4. 下发安全

- [ ] 生产数据、脚本、权限修改应附带备份步骤，避免修改动作发生异常时无法回退
