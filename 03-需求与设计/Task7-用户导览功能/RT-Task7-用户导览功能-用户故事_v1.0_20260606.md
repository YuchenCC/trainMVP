## US7.1 首次登录引导

**作为** 新用户
**我希望** 首次登录时能看到系统介绍和核心功能概览
**以便** 快速了解系统价值和基本功能

| 编号 | 规则 | 校验时机 |
|------|------|----------|
| BR7.1.1 | 首次登录时自动触发导览 | 登录时 |
| BR7.1.2 | 显示用户角色信息 | 导览开始时 |
| BR7.1.3 | 支持跳过导览 | 每个步骤 |
| BR7.1.4 | 导览完成后保存状态 | 导览结束时 |
| BR7.1.5 | 下次登录不再自动触发 | 登录时 |

- [ ] 首次登录用户能看到欢迎页面

- [ ] 导览完成后状态正确保存

## US7.2 角色专属导览

**作为** 新用户
**我希望** 能看到针对我角色的专属导览
**以便** 快速掌握与我相关的核心功能

### 业务规则

| 编号 | 规则 | 校验时机 |
|------|------|----------|
| BR7.2.1 | 根据用户角色加载对应导览 | 导览开始时 |
| BR7.2.2 | 不同角色显示不同导览内容 | 导览加载时 |
| BR7.2.3 | 支持查看其他角色导览 | 导览选择时 |
| BR7.2.4 | 记录每个角色导览的完成状态 | 导览结束时 |

### 角色导览内容

**业务BA导览**
- 需求池：查看和管理所有需求，支持筛选和搜索
- 新建需求：点击创建新需求，填写标题、描述、优先级等信息

**项目经理导览**
- 仪表盘：查看项目整体进度、各系统进度分布
- 班次日历：查看班次时间安排和容量使用情况

**火车管理员导览**
- 版本火车：管理版本火车，查看火车详情和系统配置
- 智能纳版：使用AI辅助排期建议，完成需求纳版

### 验收标准

- [ ] 根据用户角色自动加载对应导览
- [ ] 不同角色显示不同导览内容
- [ ] 导览步骤与角色功能匹配
- [ ] 支持查看其他角色导览
- [ ] 导览完成后状态正确保存
- [ ] 导览界面美观，操作流畅

## US7.3 功能点导览

**作为** 新用户
**我希望** 首次访问功能页面时能看到该页面的操作导览
**以便** 快速掌握该功能的使用方法

### 业务规则

| 编号 | 规则 | 校验时机 |
|------|------|----------|
| BR7.3.1 | 首次访问功能页面时自动触发导览 | 页面加载时 |
| BR7.3.2 | 页面右上角提供"导览"按钮 | 页面渲染时 |
| BR7.3.3 | 支持手动触发导览 | 点击按钮时 |
| BR7.3.4 | 记录每个功能页面的导览完成状态 | 导览结束时 |

### 功能点导览内容

**仪表盘导览**
- 系统筛选：切换系统，仪表盘数据跟着变。BA/PM/技术经理只能看自己所属系统，火车管理员/超管可看所有系统
- 需求统计：各状态的需求数量一目了然，点击卡片可跳转需求列表
- 待办事项：你当前需要处理的事项列表，点击可进入具体操作
- 变更率监控：绿色表示需求变更率（变更需求数/已纳版需求数）
- 紧急变更率：封板后发生的变更需要紧急变更流程，过多会影响发布质量

**需求池导览**
- 新增需求：创建新需求，支持「保存草稿」或「提交评审」
- 需求录入：在此输入需求描述，包括背景、目标和验收标准。可使用「AI智能补全」自动生成内容
- 筛选搜索：按系统、状态、关键词多维度筛选需求
- 需求列表：查看所有需求信息，点击进入详情页

**月历视图导览**
- 筛选器：选择系统和版本火车，月历自动显示对应的班次
- 班次展示：彩色横条表示班次周期，悬停查看班次详情

**版本火车导览**
- 火车列表：展示所有版本火车，点击「查看」进入详情管理班次
- 班次状态：规划中 → 进行中 → 已封板 → 已投产
- 容量使用：查看容量使用率，确保不超出上限
- 纳版管理：管理已纳版和待纳版的需求，使用AI智能纳版提升效率

**AI审查导览**
- 编辑需求：点击编辑需求内容，填写需求信息后保存，再进行AI审查
- AI审查：点击按钮，AI自动分析需求质量，包括完整性、合理性检查

### 验收标准

- [ ] 首次访问功能页面时自动触发导览
- [ ] 页面右上角提供"导览"按钮
- [ ] 点击按钮能手动触发导览
- [ ] 导览步骤与页面功能匹配
- [ ] 导览完成后状态正确保存
- [ ] 导览界面不遮挡重要内容

## 用户体验要求

### 7.1 导览界面
- 导览界面美观，不遮挡重要信息
- 高亮区域准确，动画流畅
- 引导弹窗位置合理，内容清晰
- 进度指示准确，步骤切换流畅

### 7.2 交互体验
- 控制按钮响应及时，操作准确
- 支持跳过和重新开始
- 导览步骤清晰，易于理解
- 导览完成后提供明确反馈

### 7.3 性能要求
- 导览加载时间 < 1秒
- 导览切换流畅，无明显卡顿
- 导览状态保存及时，无延迟

### 7.4 兼容性要求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 支持不同屏幕尺寸（桌面、平板、移动）
- 导览界面在不同设备上显示正常

---

## 技术要求

### 7.5 前端技术
- 使用 `react-joyride` 或 `driver.js` 实现导览功能
- 使用 React Context 或 Redux 管理导览状态
- 使用 localStorage 存储导览进度
- 支持响应式设计

### 7.6 后端技术
- 提供用户导览进度查询和更新接口
- 提供导览配置查询接口
- 支持导览内容动态配置

### 7.7 数据模型
- 用户导览进度表
- 导览配置表
- 导览步骤表
