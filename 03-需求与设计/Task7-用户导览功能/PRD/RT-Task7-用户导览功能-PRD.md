### Task 7 范围

本任务聚焦用户导览功能，帮助新用户快速了解系统核心功能和操作流程：

- **首次登录引导**: 系统整体介绍、角色定位、核心功能概览

- **随时帮助**: 快速访问导览、文档、教程

## 二、功能概述

用户导览是一个交互式引导系统，通过分步骤的可视化引导，帮助新用户快速了解版本火车需求管理系统的核心功能和操作流程，降低学习成本，提升用户上手效率。

### 2.1 业务价值

- **降低学习成本**: 新用户无需阅读冗长文档，通过交互式导览快速上手
- **提升用户体验**: 减少用户困惑，提高首次使用满意度
- **提高效率**: 用户更快掌握核心功能，减少操作错误
- **降低支持成本**: 减少用户咨询和培训需求

### 2.2 用户场景

| 角色        | 场景      | 目标             |
| --------- | ------- | -------------- |
| 新用户      | 首次登录   | 了解系统整体功能和自己的角色定位  |
| 业务BA      | 首次使用   | 掌握需求录入和跟踪的核心操作    |
| 产品经理      | 首次使用   | 掌握需求评审和版本规划的核心操作  |
| 项目经理/技术经理 | 首次使用   | 掌握进度跟踪和风险控制的核心操作  |
| 火车管理员     | 首次使用   | 掌握火车管理和纳版操作的核心功能  |
| 研发团队     | 首次使用   | 掌握需求查看和任务跟踪的核心操作  |

## 三、功能详细设计

### 3.1 首次登录引导

#### 3.1.1 触发时机
- 用户首次登录系统
- 用户未完成过任何导览

#### 3.1.2 导览内容

**步骤1: 欢迎页面**
- 欢迎语："欢迎使用版本火车需求管理系统"
- 系统价值："高效管理需求，智能规划版本"
- 角色确认："您的角色：{角色名}"
- 操作按钮：
  - "开始我的专属导览"（主按钮）
  - "跳过，直接使用"（次按钮）

**步骤2: 核心模块介绍**
- 需求池管理：录入、评审、跟踪需求
- 版本火车：创建火车、规划班次、纳版搭载
- 智能纳版：AI辅助排期建议
- 仪表盘：数据可视化、进度跟踪
- 操作按钮：
  - "下一步"
  - "跳过"

**步骤3: 快速开始**
- "开始使用"按钮
- "再次观看导览"选项
- 导览完成提示

#### 3.1.3 交互规则
- 支持跳过：每个步骤都可以跳过
- 支持重新开始：导览完成后可以重新观看
- 状态保存：记录用户导览完成状态，下次登录不再自动触发

### 3.2 角色专属导览

#### 3.2.1 触发时机
- 首次登录完成通用导览后
- 用户主动触发（帮助菜单、导览按钮）
- 用户切换角色后

#### 3.2.2 导览内容

**业务BA专属导览**

步骤1: 需求录入
- 导航到需求池 → 点击"新建需求"
- 演示填写需求标题、描述、归属系统
- 选择优先级、工作量点数、业务归属人
- 提交并查看需求编号

步骤2: 需求跟踪
- 查看需求列表
- 筛选和搜索需求
- 查看需求详情
- 了解需求状态流转

步骤3: 我的待办
- 查看待评审需求
- 发起评审操作
- 查看评审结果

步骤4: 完成导览
- "开始使用"按钮
- "再次观看导览"选项

**项目经理专属导览**

步骤1: 进度跟踪
- 查看项目整体进度
- 查看各系统进度分布
- 识别进度风险
- 查看延期预警

步骤2: 资源协调
- 查看资源分配情况
- 调整资源配置
- 协调团队工作

步骤3: 班次日历
- 查看班次日历视图
- 查看班次时间安排
- 查看班次容量使用

步骤4: 完成导览
- "开始使用"按钮
- "再次观看导览"选项

**火车管理员专属导览**

步骤1: 火车管理
- 创建版本火车
- 配置火车信息
- 添加参与系统

步骤2: 班次管理
- 创建班次
- 设置班次时间
- 查看班次日历

步骤3: 智能纳版
- 查看待纳版需求
- 使用AI排期建议
- 手动调整纳版

步骤4: 完成导览
- "开始使用"按钮
- "再次观看导览"选项

#### 3.2.3 交互规则
- 角色识别：根据用户角色自动加载对应导览
- 支持切换：用户可以查看其他角色的导览
- 进度保存：记录每个角色导览的完成状态

### 3.3 功能点导览

#### 3.3.1 触发时机
- 用户首次访问某个功能页面
- 用户主动触发（页面右上角"导览"按钮）

#### 3.3.2 导览内容

**需求录入页面导览**
- 需求标题输入
- 需求描述编辑（富文本）
- 归属系统选择
- 优先级选择
- 工作量点数输入
- 业务归属人选择
- 产品经理选择
- 依赖需求配置
- 提交按钮

**版本火车创建页面导览**
- 火车名称输入
- 火车描述输入
- 系统选择
- 容量点数配置
- 创建按钮

**智能纳版页面导览**
- 待纳版需求列表
- AI排期建议
- 手动调整功能
- 确认纳版按钮

#### 3.3.3 交互规则
- 页面级触发：首次访问页面时自动触发
- 手动触发：页面右上角提供"导览"按钮
- 状态保存：记录每个功能页面的导览完成状态

### 3.4 随时帮助

#### 3.4.1 触发方式
- 帮助菜单：页面右上角帮助按钮

#### 3.4.2 帮助内容
- 导览入口：重新观看各种导览
- 文档链接：跳转到用户手册
- 视频教程：功能操作视频（可选）
- 联系支持：技术支持联系方式

#### 3.4.3 交互规则
- 全局可用：在任何页面都可以触发
- 内容更新：支持动态配置帮助内容

## 四、交互设计

### 4.1 导览界面元素

#### 4.1.1 高亮区域
- 当前讲解的功能区域高亮显示
- 其他区域半透明遮罩
- 高亮区域添加脉冲动画

#### 4.1.2 引导弹窗
- 显示操作说明
- 显示步骤标题和内容
- 提供操作按钮（下一步、上一步、跳过、结束）
- 显示进度指示器（当前步骤/总步骤）

#### 4.1.3 进度指示
- 顶部进度条
- 步骤圆点指示器
- 当前步骤高亮显示

#### 4.1.4 控制按钮
- "下一步"：进入下一个导览步骤
- "上一步"：返回上一个导览步骤
- "跳过"：跳过当前步骤
- "结束"：结束导览
- "重新开始"：重新开始导览

### 4.2 导览控制

#### 4.2.1 导览流程
- 线性流程：按顺序执行导览步骤
- 支持跳过：可以跳过当前步骤
- 支持返回：可以返回上一个步骤
- 支持结束：可以随时结束导览

#### 4.2.2 状态管理
- 导览进行中：显示导览界面
- 导览暂停：隐藏导览界面，保留状态
- 导览完成：显示完成提示，保存状态
- 导览跳过：记录跳过状态

### 4.3 响应式设计

#### 4.3.1 屏幕适配
- 桌面端：标准导览界面
- 平板端：调整导览界面大小和位置
- 移动端：简化导览界面，优化触摸操作

#### 4.3.2 位置适配
- 自动计算导览弹窗位置
- 避免遮挡重要内容
- 支持手动调整位置

## 五、技术实现要点

### 5.1 前端技术

#### 5.1.1 导览库
- **推荐库**: `react-joyride` 或 `driver.js`
- **功能**: 步骤引导、高亮区域、进度控制
- **兼容性**: 支持主流浏览器

#### 5.1.2 状态管理
- 使用 React Context 或 Redux 管理导览状态
- 记录导览进度和完成状态
- 支持导览状态持久化

#### 5.1.3 本地存储
- 使用 localStorage 存储导览进度
- 使用 sessionStorage 存储临时导览状态
- 支持导览状态同步到后端

#### 5.1.4 响应式设计
- 使用 CSS Grid 和 Flexbox 布局
- 支持不同屏幕尺寸
- 优化移动端体验

### 5.2 后端技术

#### 5.2.1 用户配置
- 存储用户导览完成状态
- 支持导览状态查询和更新
- 提供导览配置接口

#### 5.2.2 角色识别
- 根据用户角色返回对应导览内容
- 支持角色切换
- 提供角色权限验证

#### 5.2.3 导览配置
- 支持动态配置导览内容
- 提供导览内容管理接口
- 支持导览内容版本控制

### 5.3 数据模型

#### 5.3.1 用户导览进度
```typescript
interface UserTourProgress {
  userId: string;
  role: string;
  completedTours: {
    general: boolean;          // 通用导览
    roleSpecific: boolean;     // 角色专属导览
    featureTours: Record<string, boolean>; // 功能点导览
  };
  lastTourDate: Date;
  createdAt: Date;
  updatedAt: Date;
}
```

#### 5.3.2 导览配置
```typescript
interface TourConfig {
  id: string;
  name: string;
  description: string;
  targetRoles: string[];
  trigger: 'first_login' | 'manual' | 'page_visit';
  steps: TourStep[];
  version: string;
  isActive: boolean;
  createdAt: Date;
  updatedAt: Date;

interface TourStep {
  id: string;
  target: string;              // CSS选择器或组件ID
  title: string;
  content: string;
  position: 'top' | 'bottom' | 'left' | 'right';
  disableBeacon?: boolean;
  spotlightClicks?: boolean;
  spotlightPadding?: number;
}
```

## 六、验收标准

### 6.1 功能验收

- [ ] 首次登录用户能看到欢迎页面
- [ ] 不同角色能看到对应的专属导览
- [ ] 导览步骤清晰，操作流畅
- [ ] 支持跳过和重新开始
- [ ] 导览完成后状态正确保存
- [ ] 功能点导览在首次访问页面时自动触发
- [ ] 帮助菜单提供导览入口
- [ ] 快捷键能触发帮助菜单

### 6.2 交互验收

- [ ] 导览界面美观，不遮挡重要信息
- [ ] 高亮区域准确，动画流畅
- [ ] 引导弹窗位置合理，内容清晰
- [ ] 进度指示准确，步骤切换流畅
- [ ] 控制按钮响应及时，操作准确

### 6.3 兼容性验收

- [ ] 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- [ ] 支持不同屏幕尺寸（桌面、平板、移动）
- [ ] 导览界面在不同设备上显示正常

### 6.4 性能验收

- [ ] 导览加载时间 < 1秒
- [ ] 导览切换流畅，无明显卡顿
- [ ] 导览状态保存及时，无延迟

### 6.5 可维护性验收

- [ ] 导览内容可配置化
- [ ] 导览步骤易于添加和修改
- [ ] 导览状态易于查询和管理
