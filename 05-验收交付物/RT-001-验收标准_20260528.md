## 三、验收判定规则

### 3.1 判定级别

| 级别 | 定义 | 处理方式 |
|------|------|----------|
| **致命** | 核心功能不可用，系统无法启动或崩溃 | 必须修复后重新验收 |
| **严重** | 主要功能存在缺陷，影响业务流程 | 必须修复后重新验收 |
| **一般** | 次要功能存在缺陷，不影响核心流程 | 可记录缺陷后通过，后续迭代修复 |
| **轻微** | UI问题、文案错误等 | 可记录缺陷后通过，后续迭代修复 |

### 3.2 通过标准

| 条件 | 要求 |
|------|------|
| 致命缺陷 | 0个 |
| 严重缺陷 | 0个 |
| 一般缺陷 | ≤ 3个 |
| 轻微缺陷 | ≤ 10个 |
| 测试通过率 | ≥ 95% |
