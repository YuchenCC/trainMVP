## 二、功能概述

需求池是管理所有待处理需求的中心仓库，支持业务BA录入、分类、评审和优先级管理。

---

## 三、功能详细设计

### 3.1 需求录入

| 需求项 | 详细说明 | 必填 |
|--------|----------|------|
| 需求标题 | 简洁描述需求核心内容 | ✅ |
| 需求描述 | 详细说明需求背景、目标和价值 | ✅ |
| 归属系统 | 选择参与当前版本火车的系统 | ✅ |
| 优先级 | P0/P1/P2/P3 | ✅ |
| 工作量点数 | 正整数（1-100），表示开发工作量 | ✅ |
| 业务归属人 | 该系统的业务BA | ✅ |
| 产品经理 | 该系统的产品经理 | ✅ |
| 创建人 | 实际创建该需求的用户，默认当前用户 | 自动 |
| 提出时间 | 系统自动填充 | 自动 |
| 依赖需求 | 关联其他系统的需求作为前置依赖 | ❌ |

**依赖需求规则**：
- 一个需求可配置多个依赖需求，支持跨系统依赖
- 依赖需求需选择已存在的需求（草稿及以上状态均可关联）
- 依赖关系为"前置依赖"：依赖需求未完成前，本需求不建议纳版
- 依赖需求状态为"已投产"或"已纳版"时，视为依赖已满足
- 依赖需求状态为"已取消"时，系统提示预警

### 3.2 需求分类

| 分类维度 | 可选值 |
|----------|--------|
| 需求类型 | 新功能/优化/缺陷 |
| 来源渠道 | 业务提出/用户反馈/数据分析/竞品分析 |

### 3.3 状态流转功能（10个）

#### 3.3.1 发起评审

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"草稿"，当前用户为该需求的业务归属人（BA） |
| 输入 | 需求ID |
| 输出 | 需求状态变更为"待评审" |
| 业务规则 | 1. 校验需求必填项是否完整<br>2. 校验当前用户是否为该需求的业务归属人（BA）<br>3. 记录发起评审时间和操作人 |

#### 3.3.2 评审通过

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"待评审"，当前用户为评审人（产品经理、项目经理或技术经理） |
| 输入 | 需求ID、评审意见（可选） |
| 输出 | 需求状态变更为"已就绪" |
| 业务规则 | 1. 校验当前用户是否有评审权限<br>2. 记录评审人、评审时间<br>3. 保存评审意见 |

#### 3.3.3 评审拒绝

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"待评审"，当前用户为评审人 |
| 输入 | 需求ID、拒绝原因（必填） |
| 输出 | 需求状态变更为"已拒绝" |
| 业务规则 | 1. 校验当前用户是否有评审权限<br>2. 拒绝原因必填，最多500字<br>3. 记录评审人、评审时间、拒绝原因 |

#### 3.3.4 重新编辑

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"已拒绝"，当前用户为业务归属人（BA）或产品经理 |
| 输入 | 需求ID |
| 输出 | 需求状态变更为"草稿" |
| 业务规则 | 1. 校验当前用户是否为业务归属人或产品经理<br>2. 状态变更为草稿后可编辑修改 |

#### 3.3.5 取消需求

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态不为"已取消"且不为"已投产"，当前用户为业务归属人（BA）或火车管理员 |
| 输入 | 需求ID、取消原因（必填） |
| 输出 | 需求状态变更为"已取消" |
| 业务规则 | 1. 校验当前用户是否为业务归属人或火车管理员<br>2. 取消原因必填，最多500字<br>3. 记录取消时间、操作人、取消原因<br>4. 已纳版的需求取消时，自动从版本火车移除<br>5. **已投产的需求不能取消，只能通过回滚操作撤回** |

#### 3.3.6 需求变更（普通变更）

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"已就绪"或"已纳版（开发中/SIT测试/UAT测试）"，当前用户为业务归属人（BA）或火车管理员 |
| 输入 | 需求ID、变更原因（必填） |
| 输出 | 需求状态变更为"草稿"；如已纳版则同时从版本火车移除 |
| 业务规则 | 1. 校验当前用户是否为业务归属人或火车管理员<br>2. 变更原因必填，最多500字<br>3. 记录变更时间、操作人、变更原因<br>4. 已纳版的需求变更时，自动从版本火车移除，释放团队容量<br>5. 变更后需求状态统一为"草稿"，可编辑修改<br>6. 修改完成后需重新发起评审，走完整评审流程 |

#### 3.3.7 紧急变更（封板状态）

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"已纳版（封板）"，当前用户为业务归属人（BA）或火车管理员 |
| 输入 | 需求ID、紧急变更原因（必填）、紧急程度（P0紧急/P1较紧急） |
| 输出 | 提交审批 → 审批通过后状态变更为"草稿"，从版本火车移除 |
| 业务规则 | 1. 校验当前用户是否为业务归属人或火车管理员<br>2. 紧急变更原因必填，最多500字<br>3. 紧急程度必选<br>4. 提交后进入审批流程，审批人为项目经理或火车管理员<br>5. 审批通过后：需求状态变更为"草稿"，从版本火车移除，释放容量<br>6. 审批拒绝后：维持封板状态，记录拒绝原因<br>7. 变更后需重新发起评审和纳版流程 |

#### 3.3.8 开发完成（进入SIT测试）

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"已纳版（开发中）"，当前用户为技术经理 |
| 输入 | 需求ID |
| 输出 | 需求子状态变更为"SIT测试" |
| 业务规则 | 1. 校验当前用户是否为技术经理<br>2. 记录开发完成时间<br>3. 子状态变更为"SIT测试" |

#### 3.3.9 SIT通过（进入UAT测试）

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"已纳版（SIT测试）"，当前用户为测试经理 |
| 输入 | 需求ID、SIT测试结果（可选） |
| 输出 | 需求子状态变更为"UAT测试" |
| 业务规则 | 1. 校验当前用户是否为测试经理<br>2. 记录SIT通过时间<br>3. 子状态变更为"UAT测试" |

#### 3.3.10 UAT通过（封板）

| 项目 | 说明 |
|------|------|
| 前置条件 | 需求状态为"已纳版（UAT测试）"，当前用户为业务归属人（BA） |
| 输入 | 需求ID、UAT验收结果（可选） |
| 输出 | 需求子状态变更为"封板" |
| 业务规则 | 1. 校验当前用户是否为业务归属人（BA）<br>2. 记录UAT通过时间<br>3. 子状态变更为"封板"，代码冻结<br>4. 封板后仅支持紧急变更（需审批） |

## 四、界面设计

### 4.1 需求列表页

**页面元素**：
- **系统切换区**: 下拉框切换不同系统
- **筛选区**: 按状态、创建人、版本筛选
- **需求列表**: 显示需求标题、状态、优先级、创建人、提出时间
- **操作按钮**: 根据需求状态和用户权限显示对应操作

**操作按钮显示规则**：

| 需求状态 | BA可见操作 | PM可见操作 | PROJECT_MGR | TECH_MGR | TEST_MGR | TRAIN_ADMIN |
|----------|------------|------------|-------------|----------|----------|-------------|
| 草稿 | 编辑、发起评审、取消 | 编辑 | - | - | - | 取消 |
| 待评审 | 查看、取消 | 查看、评审通过、评审拒绝 | 查看、评审通过、评审拒绝 | 查看、评审通过、评审拒绝 | - | 查看、取消 |
| 已就绪 | 查看、需求变更、取消 | 查看 | 查看 | - | - | 查看、需求变更、取消 |
| 已拒绝 | 查看、重新编辑、取消 | 查看、重新编辑 | 查看 | - | - | 取消 |
| 已纳版-开发中 | 查看、需求变更、取消 | 查看 | 查看 | 开发完成 | - | 查看、需求变更、取消 |
| 已纳版-SIT测试 | 查看、需求变更、取消 | 查看 | 查看 | - | SIT通过 | 查看、需求变更、取消 |
| 已纳版-UAT测试 | UAT通过、需求变更、取消 | 查看 | 查看 | - | - | 查看、需求变更、取消 |
| 已纳版-封板 | 查看、紧急变更、取消 | 查看 | 查看 | - | - | 查看、紧急变更、取消 |
| 已投产 | 查看 | 查看 | 查看 | - | - | 查看、回滚 |
| 已取消 | 查看 | 查看 | 查看 | - | - | 查看 |

### 4.2 需求详情页

**页面元素**：
- 基本信息（归属系统、优先级、业务归属人、产品经理等）
- 需求描述
- 依赖需求（跨系统前置依赖）
- 操作历史
- 操作按钮区（根据权限动态显示）

**依赖需求区域**：
- ✅ 绿色：依赖已满足（已纳版/已投产）
- ⚠️ 黄色：依赖未满足（草稿/待评审/已就绪/已拒绝）
- ❌ 红色：依赖已取消

### 4.3 需求评审页

**功能**：批量评审待评审需求
- 待评审需求列表（可多选）
- 需求详情预览
- 评审意见输入
- 批量通过/拒绝按钮
