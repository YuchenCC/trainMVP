## 二、设计原则

### 2.1 核心约束

1. **统一入口**：不按角色区分页面，所有角色访问同一个仪表盘地址
2. **数据跟随角色**：显示的数据内容根据用户角色动态变化
3. **不做凭空造牌**：所有展示的数据必须来自现有数据库表
4. **只读聚合查询**：所有后端 API 均为只读查询，无副作用

### 2.2 明确排除清单

| 排除项 | 原因 |
|--------|------|
| 按角色分页面路由 | 改为角色分发数据，同一页面不同数据 |
| 周/月报告一键生成 | 无报告引擎支撑 |
| 批量评审/批量操作 | 改变业务流程 |
| 仪表盘自定义布局 | MVP 阶段不实现 |

## 三、功能需求

### 3.1 系统筛选器

**功能描述**：全局系统筛选，影响所有仪表盘数据的展示范围

| 需求项 | 详细说明 |
|--------|----------|
| 位置 | 页面顶部标题栏右侧 |
| 筛选范围 | 需求统计、待办列表、班次进度 |
| 默认行为 | 选中用户所属的第一个系统 |
| 空状态 | 显示"全部系统"选项 |

**UI 设计**：
```
┌─────────────────────────────────────────────────────────────────┐
│ 🚄 版本火车需求管理系统                    [系统筛选器 ▼]       │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 统计卡片

**功能描述**：展示需求在各状态下的数量统计，点击跳转至列表页

| 需求项 | 详细说明 |
|--------|----------|
| 状态类型 | DRAFT、PENDING_REVIEW、READY、ONBOARDED、RELEASED、CANCELLED |
| 跳转行为 | 点击卡片跳转至 `/requirements?status={状态}` |
| 样式 | 左侧带状态颜色边框，hover 显示手型 |

**卡片配置**：

| 状态 | 标题 | 副标题 | 颜色 |
|------|------|--------|------|
| DRAFT | 草稿 | 待完善·发起评审 | #1890ff |
| PENDING_REVIEW | 待评审 | 评审中·待确认 | #faad14 |
| READY | 已就绪 | 待纳版·等待上车 | #52c41a |
| ONBOARDED | 已纳版 | 开发中·已上车 | #722ed1 |
| RELEASED | 已投产 | 已上线·完成交付 | #13c2c2 |

### 3.3 关键时间倒计时

**功能描述**：展示最近1个月内即将到来的关键节点

| 需求项 | 详细说明 |
|--------|----------|
| 节点类型 | 纳版截止、SIT开始、UAT开始、封板、投产 |
| 过滤条件 | 仅显示未来30天内的节点 |
| 显示内容 | 节点名称、所属班次、剩余天数 |
| 排序 | 按剩余天数升序排列 |

**关键节点定义**：

| 节点 | 字段来源 | 颜色 | 图标 |
|------|----------|------|------|
| 纳版截止 | `boardingDate` | warning (#faad14) | 🔔 |
| SIT开始 | `sitDate` | processing (#1890ff) | 🧪 |
| UAT开始 | `uatDate` | purple (#722ed1) | ✅ |
| 封板 | `lockdownDate` | error (#f5222d) | 🔒 |
| 投产 | `releaseDate` | success (#52c41a) | 🚀 |

**UI 设计**：
```
┌─────────────────────────────────────────────────────────────────┐
│ 关键时间倒计时                                                    │
├─────────────────────────────────────────────────────────────────┤
│ 🔔 纳版截止        2026年Q2第1班    还剩 3 天   2026-05-27    │
│ 🔒 封板            2026年Q2第1班    还剩 10 天  2026-06-03    │
│ 🚀 投产            2026年Q2第1班    还剩 17 天  2026-06-10    │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 待办事项列表

**功能描述**：根据用户角色展示不同的待办事项

#### 3.4.1 待办项分类

| 分类 | 描述 | 包含项 |
|------|------|--------|
| 需求审核 | 待评审的需求列表 | 需求编号、标题、系统、优先级 |
| 需求变更 | 已纳版需求被退回草稿 | 变更原因、变更时间、重新编辑入口 |
| 紧急变更 | 紧急变更申请的状态追踪 | 申请时间、紧急程度、审批状态 |

#### 3.4.2 按角色显示待办

| 角色 | 待办项 |
|------|--------|
| **BA** | 需求审核拒绝→重新编辑 + 紧急变更（拒绝→显示审批拒绝状态，通过→重新提交审核） |
| **PM** | 待评审需求 |
| **PROJECT_MGR** | 待评审需求 + 紧急变更待审批 |
| **TECH_MGR** | 待评审需求 |
| **TEST_MGR** | 紧急变更待审批 |
| **TRAIN_ADMIN** | 待评审需求 + 紧急变更待审批 |

#### 3.4.3 待办项交互

| 操作 | 描述 |
|------|------|
| 查看详情 | 点击跳转至需求详情页 |
| 重新编辑 | 跳转至需求编辑页（仅BA的审核拒绝项） |
| 审批通过 | 按钮操作紧急变更审批（仅审批人有此按钮） |
| 审批拒绝 | 按钮操作紧急变更审批（需填写驳回原因） |

**UI 设计**：
```
┌─────────────────────────────────────────────────────────────────┐
│ 待办事项                                                        │
├─────────────────────────────────────────────────────────────────┤
│ ▼ 紧急变更待审批 (3)                                             │
│   ├── REQ-2026-0101 | 订单支付功能 | P0 | 待审批    [通过][拒绝]│
│   ├── REQ-2026-0102 | 用户登录优化 | P1 | 待审批    [通过][拒绝]│
│   └── REQ-2026-0103 | 数据导出功能 | P2 | 待审批    [通过][拒绝]│
├─────────────────────────────────────────────────────────────────┤
│ ▼ 待评审需求 (5)                                                 │
│   ├── REQ-2026-0201 | 首页改版 | 普惠前端系统 | P0    [查看详情]│
│   └── ...                                                        │
├─────────────────────────────────────────────────────────────────┤
│ ▼ 紧急变更进度 (2)                                                │
│   ├── REQ-2026-0301 | P0 | 审批拒绝 | 拒绝原因：... [查看详情] │
```

## 四、数据需求

### 4.1 后端 API 清单

| API | 方法 | 描述 | 参数 |
|-----|------|------|------|
| `/api/stats/requirements` | GET | 需求统计 | systemId? |
| `/api/requirements/my-todos` | GET | 用户待办 | - |
| `/api/emergency-changes` | GET | 紧急变更列表 | systemId?, status? |
| `/api/schedules/progress` | GET | 班次进度 | systemId? |

### 4.2 待办查询逻辑

#### 4.2.1 BA 待办

```typescript
// 需求审核拒绝（需重新编辑）
pendingReviewRejected = Requirement.findMany({
  where: { status: 'REJECTED', systemId: user.systemIds }
});

// 紧急变更进度（自己发起的变更）
myEmergencyChanges = EmergencyChange.findMany({
  where: { requirement: { systemId: user.systemIds }, creatorId: user.id },
  include: { requirement: true }
});
```

#### 4.2.2 PM/TECH_MGR 待办

```typescript
// 待评审需求
pendingReviewList = Requirement.findMany({
  where: { status: 'PENDING_REVIEW' }
});
```

#### 4.2.3 PROJECT_MGR/TRAIN_ADMIN 待办

```typescript
// 待评审需求
pendingReviewList = Requirement.findMany({
  where: { status: 'PENDING_REVIEW' }
});

// 紧急变更待审批
emergencyPendingApproval = EmergencyChange.findMany({
  where: { status: 'PENDING' }
});
```

#### 4.2.4 TEST_MGR 待办

